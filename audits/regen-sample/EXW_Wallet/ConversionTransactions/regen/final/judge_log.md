### Top 5 Issues

1. **Footer/lineage column count says 13 instead of 14** — The DDL has 14 columns, the Elements table correctly has 14 rows, but the footer says "Elements: 13/13", the tier summary says "Tier 3: 13", and the lineage file says "DDL column count: 13." This is an internal inconsistency across three locations.

2. **etr_* columns could be Tier 2 rather than Tier 3** — These three columns are ETL-computed (year/month/date extraction from Occurred by the Generic Pipeline). The tier rubric defines Tier 2 as "Derived from SP code or ETL logic." While there's no SP, the Generic Pipeline IS the ETL logic. Defensible as Tier 3 but warrants acknowledgment.

3. **CryptoId values not mapped to crypto names** — 25 distinct CryptoId values are listed by frequency but not mapped to asset names (e.g., 1=BTC, 2=ETH). The review-needed sidecar flags this, but the wiki could include at least partial mappings gleaned from the EXW_TransactionsView or EXW_FactConversions code if available.

4. **No explicit Phase Gate Checklist** — The footer references "Phases: 12/14" but no Section 8 Phase Gate Checklist with P2/P3 checkboxes appears. Section 8 is "Atlassian Knowledge Sources" instead.

5. **EtoroFeePercentage values not formatted as key=value dictionary** — With only 3 distinct values (0.10, 0.50, 1.00), this column qualifies for inline key=value enumeration per the completeness checklist, but values are listed narratively in the description rather than as a structured dictionary.

### Regeneration Feedback

1. Fix the footer and lineage metadata to say 14 columns instead of 13 (three locations: wiki footer, lineage "DDL column count", lineage tier summary).
2. Consider re-tagging etr_y, etr_ym, etr_ymd as Tier 2 with source "Generic Pipeline — derived from Occurred" since they are ETL-computed.
3. Add a Phase Gate Checklist section (or rename Section 8) with explicit P2/P3 checkboxes documenting which live-data phases were completed.
4. If CryptoTypes data is queryable, include a partial CryptoId→name mapping for the top values in the CryptoId element description.
