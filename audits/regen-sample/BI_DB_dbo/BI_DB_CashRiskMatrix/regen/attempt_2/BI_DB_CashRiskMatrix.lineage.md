## ETL Pipeline

```
etoro.Trade.Position / History.ClosePosition
  → DWH_staging → SP_Dim_Position_DL_To_Synapse
  → DWH_dbo.Dim_Position [open positions as of @Date]

etoro.Trade.GetInstrument + etoro.Dictionary.Currency
  → SP_Dim_Instrument → DWH_dbo.Dim_Instrument

PriceLog/Candles → DWH_staging → SP_Fact_CurrencyPriceWithSplit_DL_To_Synapse
  → DWH_dbo.Fact_CurrencyPriceWithSplit [OccurredDateID=@DateID]

etoro.Customer + BackOffice → DWH_dbo.Dim_Customer
etoro.Dictionary.Regulation → DWH_dbo.Dim_Regulation
etoro.Dictionary.Country → DWH_dbo.Dim_Country

Fact_SnapshotEquity + Fact_CustomerUnrealized_PnL
  → DWH_dbo.V_Liabilities [DateID=@DateID]

All of the above feed SP_CashRiskMatrix(@Date):
  Step 1: #Prices — Fact_CurrencyPriceWithSplit JOIN Dim_Instrument
          (compute ConversionRate routing via cross-currency Dim_Instrument joins)
  Step 2: #Positions — Dim_Position JOIN #Prices JOIN Dim_Customer
          JOIN Dim_Regulation JOIN Dim_Country
          (open, non-settled, valid-customer, InstrumentID<100000)
          (compute signed UnitsNOP and 94 shocked Bid/Ask price columns)
  Step 3: #Stage — from #Positions, compute 47 UnitsNOP scenario columns
          (CASE on LimitRate vs scenario bid / StopRate vs scenario ask)
  Step 4: #Final — #Stage LEFT JOIN V_Liabilities (TotalCash AVG), GROUP BY
  Step 5: DELETE + INSERT → BI_DB_dbo.BI_DB_CashRiskMatrix WHERE Date=@Date
```
