## Open Issues

**ISSUE 1 (Medium priority — SP BUG)**: In `SP_AML_Periodic_Review`, the variable `@3YearsAgo_DateID` is computed using `@YearAgo_Date` instead of `@3YearsAgo_Date`. This bug affects the MR output's population filter (3-year FTD window), not the AR table directly. However, the AR table is generated by the same SP — confirm whether this date variable is used in any intermediate temp table that feeds the AR output, or if it is only used for the `#final_pop3`/`#final_pop4` MR/HR SELECT clauses.

**ISSUE 2 (Low priority — undocumented dependency)**: `BI_DB_SF_Cases_Panel` is used to compute `Has_Open_AML_SF_Case` but has no wiki entry in this repository. `Has_Open_AML_SF_Case = 1` for only 1,426 rows (0.03% of population). Document this table when processing AML SF-related batch items.

**ISSUE 3 (Low priority — column count discrepancy)**: Previous session read a DDL file (`Dataplatform/SynapseSQLPool1/...`) that no longer exists in the repo and counted 47 columns. Live `sys.columns` query confirms 45 columns. The wiki uses 45 as authoritative. If the DDL file resurfaces or SSDT project is restored, verify whether `Regulation_New` or any other columns existed in an older table version.

**ISSUE 4 (Low priority — CountryRank semantic)**: The `CountryRank` column shows unusual distribution (0=4.58M, 1=42, 2=36K, 3=22K, 4=9K). If Rank 1 = highest risk, only 42 customers at the most dangerous tier seems very low. The SP code for `#fivetran` temp table was not available (Dataplatform directory missing). Confirm: is CountryRank the GRC country tier (1=most restricted) or the count of logins from a ranked country? If rank 1 = sanctioned countries, the count of 42 is plausible.
