## 1. Business Meaning

`BI_DB_DDR_Customer_Periodic_Status` materializes **`BI_DB_dbo.SP_DDR_Customer_Periodic_Status` rolling windows** (“this week/month/quarter/year”) described in-file as *pre-aggregation … difficult computations … kept in table*.
Each **`DateID` slice** deletes/rebuilds exclusively for `@date`.

Universe originates from **`BI_DB_dbo.BI_DB_DDR_Customer_Daily_Status`** waterfall (TP → IBAN → Options → Options-MIMO → MoneyFarm) enriched with Facts + MIMO coercion (see DDR daily wiki). Periodic rows **inherit no automatic `IsValidCustomer` filter**.

Regulatory overlays:
- **`IsValidCustomer_* = 1`** matches `Fact_SnapshotCustomer` §2.2 (valid retail analytic cohort; excludes demo/non-brand labels/country exemptions). Column name here is **`IsValidCustomer`** (common analytics shorthand parallels “ValidUser”; **DDR does not rename to `IsValidUser`**).

- **`IsCreditReportValidCB_* = 1`** is the tighter CreditBureau eligibility gate per Snapshot §2.3.

Guy M changelog (2025-11-20) rewired **`IsFunded`** to examine **LAST day inside each horizon** versus “any-day-in-window”; treat earlier interpretations as stale.

**Popular Investors**: when `IsValidCustomer_* = 1` per Facts §2.2 (tier/label/country exclusions), Popular Investor personas remain in-scope for standard analytics—as with other non-demo tiers—provided the Snapshot rule set passes.

Major column belts (each repeated ×4 horizons):
**Identity/time**, **Statics** ( regulation / MiFID / player tier / geography / CB / valid cohort ), **Engagement ladders** (`IsFunded`, `FirstTimeFunded`, DDR tier counts), **MIMO-derived deposit choreography** (global + modality splits + Options/MoneyFarm add-ons ), **Operational metadata**.

## 2. Business Logic

### 2.1 Grain & Keys

**What**: Exactly **one reconstructed row per (`RealCID`, `DateID`)** after nested `GROUP BY`s.

### 2.2 Period Boundaries — Verbatim Expressions (PHASE 9)

**What**: Window edges taken exactly from **`BI_DB_dbo.SP_DDR_Customer_Periodic_Status`** header declarations.

```
declare @weekstart DATE = DATEADD(week, DATEDIFF(ww, 0, @date), -1)
declare @monthstart date = DATEADD(month, DATEDIFF(mm, 0, @date), 0)
declare @quarterstart date = DATEADD(qq, DATEDIFF(qq, 0, @date), 0)
declare @yearstart date = DATEADD(yy, DATEDIFF(yy, 0, @date), 0)
declare @weekstartInt  = CAST(CONVERT(varchar(8), DATEADD(week, DATEDIFF(ww, 0, @date), -1), 112) AS INT)
... (matching INT twins for month/quarter/year shown in wiki §4 Elements for each `*DateID` column)
```

**Rules**:
- **`ThisWeek`** filter = `DateID BETWEEN @weekstartInt AND @dateInt`
- **`ThisMonth`** = `DateID BETWEEN @monthstartInt AND @dateInt`
- **`ThisQuarter`** = `DateID BETWEEN @quarterstartInt AND @dateInt`
- **`ThisYear`** = `DateID BETWEEN @yearstartInt AND @dateInt`

### 2.3 Static Attributes vs Activity Metrics

**What**: INNER CTE distinguishes **daily snapshot carry-forward** versus **SUM/MAX intra-period pulses**.

### 2.4 `IsValidCustomer` / `IsCreditReportValidCB` (Canonical Fact rules)

Below reproduces **`DWH_dbo.Fact_SnapshotCustomer.md` §§2.2–2.3 verbatim** — periodic columns simply replay these flags on **`@date` daily slice** aggregated per window.

```
IsValidCustomer = 1 IF:
  PlayerLevelID <> 4 (not demo)
  AND LabelID NOT IN (30, 26) (not internal/excluded label)
  AND CountryID <> 250 (not blocked country)
ELSE 0
```

```
IsCreditReportValidCB = 1 IF:
  NOT (PlayerLevelID = 4 AND AccountTypeID <> 2)  (not non-real demo)
  AND LabelID NOT IN (26, 30)
  AND NOT (CountryID = 250 AND CID NOT IN (3400616, 10526243))
ELSE 0
```

⚠ Periodic table **DOES NOT APPLY** either filter implicitly — BI reporting must **`WHERE`** explicitly.

## 3. Query Advisory

### 3.1 Distribution & CCI Discipline

Filter **`DateID` first**, then **`RealCID`** selective lookups (HASH aligns with daily DDR).

### 3.2 Common Patterns

| Question | Approach |
|-----------|----------|
| Valid retail movers | ```sql WHERE DateID=@d AND ActiveTraded_ThisMonth>0 AND IsValidCustomer_ThisMonth=1``` |
| Credit-bureau-compliant subset | Append `AND IsCreditReportValidCB_ThisWeek=1` |
| Understand funded pulse | Inspect `IsFunded_<Period>` with Nov-2025 “last-day-in-window” behavior |

### 3.3 JOINs

| Target | Clause | Purpose |
|--------|---------|---------|
| `BI_DB_dbo.BI_DB_DDR_Customer_Daily_Status` | (`RealCID`,`DateID=@anchor`) | Drill down to daily spine |
| `DWH_dbo.Dim_Country` | `CountryID_<Period>` | Region labels |
| `DWH_dbo.Dim_Regulation` | `RegulationID_<Period>` | Licensing |

### 3.4 Gotchas

1. **`SUM` on IDs** redundant but present — regulate joins on ints carefully.
2. **YTD segmentation bug suspicion** (`Portfolio_Only_ThisYear`, `BalanceOnlyAccount_ThisYear`).
3. **Options/MoneyFarm** slices younger than core ladder — chronological feature adoption per SP changelog.
4. **UC vs Synapse DDL** — ingestion partition columns **`etr_*` exist ONLY in UC mirror**.
