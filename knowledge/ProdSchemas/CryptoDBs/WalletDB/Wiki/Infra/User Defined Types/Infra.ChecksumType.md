## 1. Business Meaning

Infra.ChecksumType is a table-valued parameter (TVP) type that enables batch insertion of cryptographic checksum records into the Infra.Checksum table. It mirrors the full column structure of Infra.Checksum (excluding the identity Id column), carrying the entity identifier, checksum category, JWT signature, and creation timestamp.

This type exists to support efficient bulk writes to the checksum store. When the application needs to record data integrity signatures for multiple wallet entities in a single operation - for example, after processing a batch of wallet balance changes - it populates a ChecksumType TVP and passes it to Infra.InsertChecksumList. This avoids per-row round-trips and leverages set-based INSERT with idempotent duplicate detection.

The type is used exclusively by the Infra.InsertChecksumList stored procedure, which performs a LEFT JOIN against existing records to insert only checksums that do not already exist for the given ChecksumId + ChecksumType combination.

## 2. Business Logic

### 2.1 Idempotent Batch Insert Pattern

**What**: The TVP carries all fields needed for checksum records, enabling the consuming procedure to perform idempotent batch inserts.

**Columns/Parameters Involved**: `ChecksumId`, `ChecksumType`, `Signature`, `Created`

**Rules**:
- All four fields are NOT NULL, ensuring complete records are always provided
- The consuming procedure (InsertChecksumList) uses a LEFT JOIN to skip records where ChecksumId + ChecksumType already exists in Infra.Checksum
- The Created timestamp is caller-provided (not server-defaulted), allowing the application to control the exact audit timestamp

**Diagram**:
```
Application batch
    |
    v
[ChecksumType TVP]  -->  InsertChecksumList SP
    (N rows)                     |
                                 v
                          LEFT JOIN Infra.Checksum
                          on ChecksumId + ChecksumType
                                 |
                                 v
                          INSERT only new rows
                          (skip duplicates)
```

## 8. Sample Queries

### 8.1 Declare and populate for batch insert
```sql
DECLARE @Checksums Infra.ChecksumType
INSERT INTO @Checksums (ChecksumId, ChecksumType, Signature, Created)
VALUES
    ('12345', 'Wallet', 'eyJhbGciOiJSUzI1NiIs...', GETUTCDATE()),
    ('67890', 'WalletPool', 'eyJhbGciOiJSUzI1NiIs...', GETUTCDATE())

EXEC Infra.InsertChecksumList @Checksums = @Checksums
```

### 8.2 Populate from wallet data for bulk signing
```sql
DECLARE @Checksums Infra.ChecksumType
INSERT INTO @Checksums (ChecksumId, ChecksumType, Signature, Created)
SELECT CAST(w.WalletId AS nvarchar(64)), 'Wallet', 'computed-jwt-here', GETUTCDATE()
FROM Wallet.Wallets w WITH (NOLOCK)
WHERE w.WalletId BETWEEN 1000 AND 2000
```

### 8.3 Preview what would be inserted (dry run)
```sql
DECLARE @Checksums Infra.ChecksumType
INSERT INTO @Checksums (ChecksumId, ChecksumType, Signature, Created)
VALUES ('12345', 'Wallet', 'eyJhbGciOiJSUzI1NiIs...', GETUTCDATE())

SELECT ck.ChecksumId, ck.ChecksumType,
       CASE WHEN c.Id IS NOT NULL THEN 'SKIP (exists)' ELSE 'WILL INSERT' END AS Action
FROM @Checksums ck
    LEFT JOIN Infra.Checksum c WITH (NOLOCK)
        ON c.ChecksumId = ck.ChecksumId AND c.ChecksumType = ck.ChecksumType
```
