# Infra.InsertChecksum

## 1. Business Meaning

Infra.InsertChecksum is the single-record insertion procedure for the data integrity checksum system. It stores a cryptographic JWT signature for one wallet entity (wallet, wallet pool, or external address) into the Infra.Checksum table.

This procedure exists as the atomic unit of checksum storage. It is the entry point when the application generates a new integrity signature for a single entity - for example, after creating a new wallet or modifying wallet pool data. Without it, individual checksum writes would require raw INSERT statements without the built-in duplicate safety.

The procedure is called by the application service layer after generating a JWT signature for a wallet entity. It uses a NOT EXISTS guard to silently skip the insert if a checksum already exists for the same ChecksumId + ChecksumType combination - this makes it safe for retry scenarios and idempotent processing. Errors are caught and re-raised with the original message and severity.

## 2. Business Logic

### 2.1 Idempotent Single Insert

**What**: The procedure guarantees that duplicate checksums are never written for the same entity.

**Columns/Parameters Involved**: `@ChecksumId`, `@ChecksumType`, `@Signature`, `@Created`

**Rules**:
- Before inserting, checks WHERE NOT EXISTS for the same ChecksumId + ChecksumType in Infra.Checksum
- If a matching record exists, the INSERT silently does nothing (no error, no update)
- If no match exists, inserts the new checksum record
- The NOT EXISTS check is consistent with the UQ_ChecksumId_ChecksumType unique index on the target table
- No UPDATE path exists - checksums are immutable once written

### 8.3 Verify idempotency - second call is silently skipped
```sql
-- First call inserts the record
EXEC Infra.InsertChecksum @ChecksumId = '999', @ChecksumType = 'Wallet',
    @Signature = 'jwt-token-here', @Created = '2026-04-15T00:00:00'
-- Second call with same key does nothing (no error)
EXEC Infra.InsertChecksum @ChecksumId = '999', @ChecksumType = 'Wallet',
    @Signature = 'different-jwt-token', @Created = '2026-04-15T01:00:00'
-- Only the first signature is stored
SELECT * FROM Infra.Checksum WITH (NOLOCK) WHERE ChecksumId = '999'
```
