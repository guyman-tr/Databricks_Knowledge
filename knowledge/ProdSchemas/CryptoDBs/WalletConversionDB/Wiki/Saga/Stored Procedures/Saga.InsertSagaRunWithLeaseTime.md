## 1. Business Meaning

InsertSagaRunWithLeaseTime is the saga creation procedure - the single entry point for initiating a new saga orchestration. It atomically creates three related records in a single transaction: the saga run (SagaRuns), its initial status history entry (SagaRunStatuses with Start status), and a distributed lease (SagaLeaseTime) granting the calling worker exclusive ownership.

This is the most critical write procedure in the saga system. Every CryptoToFiatSaga conversion begins here. The transactional guarantee ensures a saga is either fully initialized (with run + status + lease) or not created at all.

---

## 2. Business Logic

### 2.1 Transactional Three-Table Insert

**What**: Creates SagaRuns row, SagaRunStatuses row, and SagaLeaseTime row in a single transaction with deduplication.

**Columns/Parameters Involved**: All parameters

**Rules**:
1. BEGIN TRANSACTION
2. INSERT INTO SagaRuns WHERE NOT EXISTS (SELECT 1 FROM SagaRuns WHERE SagaKey = @SagaKey) - dedup check
3. Get @SagaRunsId = SCOPE_IDENTITY()
4. If @SagaRunsId IS NULL -> RAISERROR 'SagaKey already exists'
5. INSERT INTO SagaRunStatuses (SagaRunId, Created, SagaStatusTypeId) - initial status history
6. INSERT INTO SagaLeaseTime (SagaKey, InstanceId, LastUpdaed, LeaseTime) - distributed lease with DATEADD(MILLISECOND, @LeaseTimeInMs, GETUTCDATE())
7. SELECT CAST(1 AS BIT) LeaseStatus
8. COMMIT TRANSACTION
- On error: ROLLBACK if @@trancount=1, COMMIT if @@trancount>1 (nested transaction handling), THROW + return LeaseStatus=0

### 2.2 Idempotency via SagaKey Deduplication

**What**: Prevents duplicate saga creation for the same business operation.

**Columns/Parameters Involved**: `@SagaKey`

**Rules**:
- Uses NOT EXISTS check before INSERT (application-level dedup)
- SagaRuns has a UNIQUE index on SagaKey (database-level enforcement)
- If SagaKey already exists: raises error with the SagaKey value in the message
- This prevents duplicate conversion requests from creating multiple saga runs

## 8. Sample Queries

### 8.1 Create a new saga run
```sql
EXEC Saga.InsertSagaRunWithLeaseTime
    @SagaName = 'CryptoToFiatSaga',
    @SagaKey = NEWID(),
    @InstanceId = 'EC3FDAF2-C03E-47A1-8025-E42050D8EA4A',
    @SagaStatusTypeId = 1,
    @LeaseTimeInMs = 300000,
    @CorrelationId = NEWID(),
    @AdditionalData = '{"SagaRequest": "..."}'
```

### 8.2 Verify the saga was created
```sql
SELECT sr.Id, sr.SagaName, sr.SagaKey, sr.SagaStatusTypeId, sl.InstanceId, sl.LeaseTime
FROM Saga.SagaRuns sr WITH (NOLOCK)
INNER JOIN Saga.SagaLeaseTime sl WITH (NOLOCK) ON sl.SagaKey = sr.SagaKey
WHERE sr.SagaKey = @SagaKey
```

### 8.3 Verify initial status was logged
```sql
SELECT rs.SagaRunId, st.Name AS Status, rs.Created
FROM Saga.SagaRunStatuses rs WITH (NOLOCK)
INNER JOIN Saga.SagaStatusTypes st WITH (NOLOCK) ON st.Id = rs.SagaStatusTypeId
WHERE rs.SagaRunId = (SELECT Id FROM Saga.SagaRuns WITH (NOLOCK) WHERE SagaKey = @SagaKey)
```
