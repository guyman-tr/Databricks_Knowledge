# C2F.InsertFiatTransaction

> Records the actual fiat transaction for a completed conversion, storing locked rates, final amounts, account routing, and fee details with deduplication protection.

## 1. Business Meaning

InsertFiatTransaction records the fiat-side completion of a conversion - the actual amount credited, the locked exchange rates, and the destination account. This is the final data step in the conversion pipeline, confirming that fiat was delivered. It includes deduplication via LEFT JOIN anti-pattern (only inserts if no fiat transaction exists for the conversion).

Called by the saga orchestrator after the fiat credit step completes successfully. The @Details parameter contains the reference ID generated by GenerateUniqueClientLoadReferenceId.

---

## 2. Business Logic

### 2.1 Dedup via LEFT JOIN Anti-Pattern

**What**: Uses INSERT...SELECT with LEFT JOIN to atomically check and insert.

**Columns/Parameters Involved**: `@CorrelationId`

**Rules**:
- INSERT FROM Conversions LEFT JOIN FiatTransactions ON ConversionId WHERE ft.Id IS NULL
- If a FiatTransaction already exists for this conversion, the WHERE ft.Id IS NULL prevents re-insertion
- This handles retry scenarios where the saga might re-attempt the fiat step

## 8. Sample Queries

### 8.1 Record a fiat transaction
```sql
EXEC C2F.InsertFiatTransaction
    @CorrelationId = 'BD637018-99FC-40AD-A466-773D7274F16C',
    @CryptoToFiatRate = 1.3564, @FiatToUsdRate = 1.0, @CryptoToUsdRate = 1.3564,
    @FiatAmount = 135.64, @UsdAmount = 135.64,
    @RateTimestamp = '2026-04-15T09:04:00.000', @AccountId = '1564157',
    @Details = 'C2F42756316', @ConversionFeeAmount = 0.13564
```

### 8.2 Verify fiat transaction was recorded
```sql
SELECT ft.* FROM C2F.FiatTransactions ft WITH (NOLOCK)
INNER JOIN C2F.Conversions c WITH (NOLOCK) ON c.Id = ft.ConversionId
WHERE c.CorrelationId = @CorrelationId
```

### 8.3 Full conversion summary after fiat completion
```sql
SELECT c.Gcid, c.CryptoAmount, eft.FiatAmount AS EstimatedFiat, ft.FiatAmount AS ActualFiat, ft.ConversionFeeAmount
FROM C2F.Conversions c WITH (NOLOCK)
INNER JOIN C2F.EstimatedFiatTransactions eft WITH (NOLOCK) ON eft.ConversionId = c.Id
INNER JOIN C2F.FiatTransactions ft WITH (NOLOCK) ON ft.ConversionId = c.Id
WHERE c.CorrelationId = @CorrelationId
```
