## 1. Business Meaning

dbo.tblaff_Tier2Members stores the affiliate hierarchy relationships that power the multi-tier commission model. Each row links a parent affiliate (AffiliateID) to a child affiliate (NewMemberID), creating the tree structure used to cascade commissions. When a customer referred by NewMemberID generates revenue, commissions flow upward through the tier chain defined in this table.

This table is essential for the multi-tier commission engine. Without it, only Tier 1 (direct referral) commissions could be calculated. The tier chain enables 5-level deep commission distribution where each level receives a configurable percentage of the generated commission.

The `AddTier2Member` procedure creates new relationships (WRITER). Trigger-based RI ensures both AffiliateID and NewMemberID reference valid affiliates. The table is system-versioned (temporal) with history in `History.tblaff_Tier2Members`, providing full audit trail of tier relationship changes. Contains ~23K active relationships.

## 2. Business Logic

### 2.1 Affiliate Tier Hierarchy

**What**: Multi-level parent-child affiliations enabling cascading commissions.

**Columns/Parameters Involved**: `AffiliateID`, `NewMemberID`, `SubAffiliateID`

**Rules**:
- `AffiliateID` is the PARENT (referring) affiliate who recruited `NewMemberID`
- `NewMemberID` is the CHILD (recruited) affiliate
- AffiliateID=1 appears frequently as a parent - likely the root/house affiliate
- Multiple tiers are constructed by chaining: if A recruited B and B recruited C, then A->B and B->C exist as separate rows
- SubAffiliateID tracks the specific sub-affiliate tag used for the recruitment
- Triggers enforce referential integrity: NewMemberID MUST exist in tblaff_Affiliates

**Diagram**:
```
Root Affiliate (ID=1)
  +-- Affiliate 61739 (NewMemberID=61739, parent=1)
  |     +-- Affiliate 61740 (NewMemberID=61740, parent=61739)
  +-- Affiliate 61737 (NewMemberID=61737, parent=1)
        +-- Affiliate 61738 (NewMemberID=61738, parent=61737)
```

## 8. Sample Queries

### 8.1 Find all sub-affiliates for a given parent
```sql
SELECT NewMemberID, SubAffiliateID
FROM dbo.tblaff_Tier2Members WITH (NOLOCK)
WHERE AffiliateID = @ParentAffiliateID
ORDER BY AffiliateLevelID
```

### 8.2 Find the full tier chain for an affiliate
```sql
;WITH TierChain AS (
    SELECT AffiliateID, NewMemberID, 1 AS TierLevel
    FROM dbo.tblaff_Tier2Members WITH (NOLOCK)
    WHERE NewMemberID = @AffiliateID
    UNION ALL
    SELECT t.AffiliateID, t.NewMemberID, tc.TierLevel + 1
    FROM dbo.tblaff_Tier2Members t WITH (NOLOCK)
    JOIN TierChain tc ON t.NewMemberID = tc.AffiliateID
    WHERE tc.TierLevel < 5
)
SELECT * FROM TierChain ORDER BY TierLevel
```

### 8.3 Count sub-affiliates per parent
```sql
SELECT AffiliateID, COUNT(*) AS SubAffiliateCount
FROM dbo.tblaff_Tier2Members WITH (NOLOCK)
GROUP BY AffiliateID
ORDER BY SubAffiliateCount DESC
```
