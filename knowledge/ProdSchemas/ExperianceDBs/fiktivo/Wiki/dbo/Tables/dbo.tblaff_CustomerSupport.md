## 1. Business Meaning

dbo.tblaff_CustomerSupport records customer events - primarily first-time deposits - that are managed by account managers and eligible for affiliate commission processing. Each row represents a customer interaction tracked in the affiliate system with associated commission data, linking the customer to their assigned manager and the referring affiliate.

This table enables the affiliate system to track and compensate for customer activity that occurs under the care of specific account managers. The commission column allows per-event commission calculation, while the FTD fields track first-time deposit activity for CPA-based payouts.

Data is created when customers managed by the support team trigger trackable events (deposits, FTDs). The table contains ~808K records through 2012. Two event types exist: Type 1 (99.8% of records - standard events) and Type 2 (0.2% - a secondary/escalated event category). Most records are marked as "Real" accounts (not demo).

## 2. Business Logic

### 2.1 Event Type Classification

**What**: Two types of customer support events tracked for commission purposes.

**Columns/Parameters Involved**: `Type`

**Rules**:
- Type "1": Standard customer support event (805K records, 99.8%). The dominant type for regular customer activity tracking.
- Type "2": Secondary event type (1,790 records, 0.2%). Likely represents a different event category such as deposit-specific events or escalated interactions.

### 2.2 Manager-Based Customer Assignment

**What**: Each event is assigned to a specific account manager, enabling per-manager commission tracking.

**Columns/Parameters Involved**: `ManagerID`, `Commission`, `CID`

**Rules**:
- ManagerID is stored as nchar(10) with trailing spaces (e.g., "400       "), suggesting legacy fixed-width data format
- Commission can be 0 (no commission for this event) or a positive amount (e.g., $61.36, $204.53)
- Each event links a customer (CID) to a manager for accountability and compensation tracking

### 2.3 FTD Tracking

**What**: First-time deposit detection and amount capture for CPA commission triggers.

**Columns/Parameters Involved**: `IsFTD`, `FTDAmount`

**Rules**:
- `IsFTD = 1`: Customer's first deposit - triggers CPA commission logic
- `IsFTD = 0`: Not a first deposit
- `FTDAmount` captures the deposit amount in account currency (values from $50 to $2,556 in sample)
- FTD identification is critical for CPA-based affiliate compensation models

## 8. Sample Queries

### 8.1 FTD events by manager with commission totals
```sql
SELECT RTRIM(ManagerID) AS ManagerID, COUNT(*) AS EventCount,
       SUM(CASE WHEN IsFTD = 1 THEN 1 ELSE 0 END) AS FTDCount,
       SUM(Commission) AS TotalCommission, SUM(FTDAmount) AS TotalFTDAmount
FROM dbo.tblaff_CustomerSupport WITH (NOLOCK)
GROUP BY RTRIM(ManagerID)
ORDER BY TotalCommission DESC
```

### 8.2 Events by type
```sql
SELECT [Type], COUNT(*) AS EventCount,
       SUM(Commission) AS TotalCommission,
       AVG(FTDAmount) AS AvgFTDAmount
FROM dbo.tblaff_CustomerSupport WITH (NOLOCK)
GROUP BY [Type]
ORDER BY EventCount DESC
```

### 8.3 High-value FTD events with affiliate attribution
```sql
SELECT TOP 10 ID, CID, FTDAmount, Commission, SerialID, SubSerialID,
       RTRIM(ManagerID) AS ManagerID, Occurred
FROM dbo.tblaff_CustomerSupport WITH (NOLOCK)
WHERE IsFTD = 1 AND FTDAmount > 1000
ORDER BY FTDAmount DESC
```
