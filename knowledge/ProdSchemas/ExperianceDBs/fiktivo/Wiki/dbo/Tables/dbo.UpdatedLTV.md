## 1. Business Meaning

This table stores periodic snapshots of customer Lifetime Value (LTV) calculations. LTV is a critical financial metric that estimates the total revenue a customer will generate over their entire relationship with the platform, combining actual historical commissions with predictive future value models.

For the affiliate business, LTV determines the long-term profitability of customer acquisitions. High-LTV customers justify higher CPA (Cost Per Acquisition) payments to affiliates, while low-LTV patterns may trigger affiliate type reclassification or commission adjustments.

Each row represents one LTV calculation snapshot for one customer at a specific UpdateDate. The same customer may have multiple rows as LTV is recalculated periodically (the data shows calculations from 2013, indicating this is a long-running batch process). The table stores both gross (before platform costs) and net (after costs) variants for both past actuals and future projections.

## 2. Business Logic

### 2.1 LTV Decomposition Model

**What**: LTV is split into four components: past gross, past net, future gross, and future net.

**Columns/Parameters Involved**: `PastGrossCommissions`, `PastNetCommissions`, `FutureGrossLTV`, `FutureNetLTV`, `TotalGrossLTV`, `TotalNetLTV`

**Rules**:
- TotalGrossLTV = PastGrossCommissions + FutureGrossLTV
- TotalNetLTV = PastNetCommissions + FutureNetLTV
- Past values are actual realized commissions from closed positions
- Future values are modeled/predicted based on customer behavior patterns
- When FutureGrossLTV = 0 and FutureNetLTV = 0, the model predicts no further activity from this customer (churned)
- Net values can be lower than gross values due to platform costs, chargebacks, or bonus deductions

### 2.2 Customer ID Remapping

**What**: Tracks both current and original customer IDs for migrated accounts.

**Columns/Parameters Involved**: `CID`, `OriginalCID`, `SerialID`

**Rules**:
- CID is the customer's current platform identifier
- OriginalCID preserves the pre-migration ID for cross-system reconciliation
- SerialID links to the affiliate tracking mechanism that acquired this customer

## 8. Sample Queries

### 8.1 Get latest LTV for a customer
```sql
SELECT TOP 1 CID, UpdateDate, TotalGrossLTV, TotalNetLTV, FutureGrossLTV
FROM dbo.UpdatedLTV WITH (NOLOCK)
WHERE CID = 1802216
ORDER BY UpdateDate DESC
```

### 8.2 Find high-LTV customers for an affiliate
```sql
SELECT u.CID, u.TotalNetLTV, u.PastNetCommissions, u.FutureNetLTV
FROM dbo.UpdatedLTV u WITH (NOLOCK)
WHERE u.SerialID = 22765
  AND u.TotalNetLTV > 100
ORDER BY u.TotalNetLTV DESC
```

### 8.3 Identify churned vs active customers
```sql
SELECT
    CASE WHEN FutureGrossLTV = 0 THEN 'Churned' ELSE 'Active' END AS Status,
    COUNT(*) AS CustomerCount,
    AVG(TotalNetLTV) AS AvgLTV
FROM dbo.UpdatedLTV WITH (NOLOCK)
GROUP BY CASE WHEN FutureGrossLTV = 0 THEN 'Churned' ELSE 'Active' END
```
