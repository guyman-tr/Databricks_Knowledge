## 1. Business Meaning

This table defines the categories used to organize marketing banners and media assets in the affiliate platform. Each category represents a grouping such as "Text Links", "Landing Pages", "Widgets", "Video", or language-specific campaign sets (e.g., "English_Social_Woman", "German_VIP 6000").

Without this table, affiliate managers could not organize the hundreds of marketing assets into browsable groups. Categories appear in the banner management UI and in the affiliate-facing banner selection tool, enabling affiliates to find relevant creative assets for their market.

Categories are managed by admin users with Categories_* permissions (from tblaff_User). Deleting a category cascade-deletes related entries in tblaff_AffiliateTypeCategories (via trigger). The table includes a mix of active campaign categories, internal-use categories ("INTERNAL USE!", "NOT TO USE"), and test data entries generated by automated testing.

## 2. Business Logic

### 2.1 Category Lifecycle with Cascade Integrity

**What**: Categories enforce referential integrity with tblaff_AffiliateTypeCategories through triggers.

**Columns/Parameters Involved**: `CategoryID`

**Rules**:
- DELETE trigger (tblaff_Categories_DTrig): cascade-deletes all matching rows in tblaff_AffiliateTypeCategories
- UPDATE trigger (tblaff_Categories_UTrig): prevents CategoryID updates if dependent rows exist in tblaff_AffiliateTypeCategories (RAISERROR 60090)
- This ensures no orphaned category-to-affiliate-type mappings

## 8. Sample Queries

### 8.1 List all active categories (excluding test data)
```sql
SELECT CategoryID, CategoryName
FROM dbo.tblaff_Categories WITH (NOLOCK)
WHERE CategoryName NOT LIKE 'new%'
  AND CategoryName NOT LIKE 'Category %'
  AND CategoryName NOT LIKE 'test%'
ORDER BY CategoryName
```

### 8.2 Find categories with their affiliate type assignments
```sql
SELECT c.CategoryID, c.CategoryName, atc.AffiliateTypeID
FROM dbo.tblaff_Categories c WITH (NOLOCK)
LEFT JOIN dbo.tblaff_AffiliateTypeCategories atc WITH (NOLOCK) ON c.CategoryID = atc.CategoryID
ORDER BY c.CategoryName
```

### 8.3 Count banners per category
```sql
SELECT c.CategoryID, c.CategoryName, COUNT(b.BannerID) AS BannerCount
FROM dbo.tblaff_Categories c WITH (NOLOCK)
LEFT JOIN dbo.tblaff_Banners b WITH (NOLOCK) ON c.CategoryID = b.CategoryID
GROUP BY c.CategoryID, c.CategoryName
ORDER BY BannerCount DESC
```
