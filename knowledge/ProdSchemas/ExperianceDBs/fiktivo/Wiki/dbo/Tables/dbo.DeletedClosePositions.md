## 1. Business Meaning

This table serves as a soft-delete archive for closed trading positions that were removed from the active `dbo.ClosedPositionsTbl`. When the system determines that certain closed positions should be purged from the active processing table (e.g., after full commission calculation, duplicate detection, or data correction), the records are moved here rather than permanently deleted.

Preserving deleted positions is essential for financial audit trails and reconciliation. Regulatory requirements mandate that all trading activity records be retained even if they are no longer active in the commission processing pipeline.

The structure mirrors `dbo.ClosedPositionsTbl` exactly (same columns in the same order), with an additional `RN` column that stores the row number from the deletion batch. The table is currently empty (0 rows), suggesting this environment has not run a position cleanup cycle.

---

## 2. Business Logic

### 2.1 Position Processing Pipeline Flags

**What**: Two boolean flags track position progress through the update and commission processing pipeline.

**Columns/Parameters Involved**: `FinishedUpdating`, `FinishedProcessing`

**Rules**:
- FinishedUpdating=1 means the position's financial data (commission, net profit, lots) has been fully calculated from the external trading system
- FinishedProcessing=1 means affiliate commission attribution has been completed for this position
- A position is "fully done" when both flags are 1
- Positions were deleted from ClosedPositionsTbl only after reaching a terminal state
