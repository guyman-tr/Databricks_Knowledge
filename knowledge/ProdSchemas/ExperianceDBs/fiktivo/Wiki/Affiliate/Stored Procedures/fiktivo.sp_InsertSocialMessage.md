## 1. Business Meaning

sp_InsertSocialMessage records social media sharing events generated by the trading platform's social features. When a user shares trading activity (such as opening a position, winning a prize, or posting user-generated content) to a social network, this procedure persists the details of that message for tracking and analytics purposes.

The message captures the full context of the social share: who shared it, what was shared (instrument, position details, gains), which social network it was posted to, and contextual metadata such as country, language, and risk profile. This data supports affiliate commission models tied to social engagement and provides analytics on which trading events drive the most social sharing.

The procedure uses a straightforward INSERT with all parameters mapped directly to table columns, returning the newly generated identity value as the message ID. The ten optional fields (@Optional1 through @Optional10) provide extensibility for future message attributes without schema changes.

## 8. Sample Queries

### 8.1 Insert a basic social message
```sql
DECLARE @NewID INT
EXEC @NewID = fiktivo.sp_InsertSocialMessage
    @MessageTypeID = 1, @Network = 2, @GCID = 100001, @CID = 50001,
    @PID = 10, @PositionID = 999999, @IsBuy = 1, @InstrumentID = 42,
    @CountryID = 1, @LocationID = 5, @TimpePeriodType = 1, @RiskType = 2,
    @Leverage = 10, @PresetMessageID = 0, @Rank = 3, @FromCID = 0,
    @FromUserName = N'TraderJoe', @InstrumentName = N'EUR/USD',
    @UserGeneratedContent = N'Great trade!', @Gain = 150.50,
    @TotalPrizes = 0, @Prize = 0, @LanguageID = 1, @IsReal = 1
SELECT @NewID AS NewMessageID
```

### 8.2 View recent social messages
```sql
SELECT TOP 100 *
FROM dbo.tblaff_SocialMessages WITH (NOLOCK)
ORDER BY 1 DESC
```

### 8.3 Count social messages by network and message type
```sql
SELECT Network, MessageTypeID, COUNT(*) AS MessageCount
FROM dbo.tblaff_SocialMessages WITH (NOLOCK)
GROUP BY Network, MessageTypeID
ORDER BY MessageCount DESC
```
