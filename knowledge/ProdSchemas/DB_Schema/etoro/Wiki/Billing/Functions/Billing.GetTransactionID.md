## 1. Business Meaning

`Billing.GetTransactionID` is a thin scalar wrapper over `Billing.GenTransactionID` (a view that generates a random 6-character hex token using NEWID()). The function exists to make token generation callable as a scalar value expression (e.g., `SELECT Billing.GetTransactionID()`) inside stored procedures, rather than requiring a subquery or SELECT FROM the view.

The function is used by billing payment procedures to generate short unique identifiers for new payment actions. These 6-character tokens are embedded in payment records as lightweight transaction identifiers. The underlying view `Billing.GenTransactionID` generates values by extracting 6 hex characters from a GUID, giving approximately 16.7 million possible values (not globally unique at high volume - see GenTransactionID documentation for collision risk notes).

Data flow: billing stored procedures call `Billing.GetTransactionID()` at the start of a payment operation to generate a token; the token is stored in the payment record. The same token generation logic is accessible via the view directly, but the function form is preferred in procedural code.

## 2. Business Logic

### 2.1 Token Generation Delegation

**What**: The function is a pure delegation wrapper - it calls the view and returns its single value.

**Columns/Parameters Involved**: (none - no input parameters)

**Rules**:
- No parameters. Called as `Billing.GetTransactionID()`.
- Internally: `SELECT @Value = Value FROM Billing.GenTransactionID`
- The view generates a new NEWID()-based token on each call, so each function invocation returns a different value.
- Token format: CHAR(6), uppercase hex characters (0-9, A-F), example: `"A4F1B3"`
- See `Billing.GenTransactionID` for the full algorithm: SUBSTRING(CONVERT(VARCHAR(36), NEWID()), 30, 6).

**Diagram**:
```
Billing.GetTransactionID()
  -> SELECT Value FROM Billing.GenTransactionID
       -> SUBSTRING(CONVERT(VARCHAR(36), NEWID()), 30, 6)
            -> Returns 6-char hex token (e.g., "A4F1B3")
```
