## 1. Business Meaning

`Billing.PaymentByWireTransferAdd` is the bank wire transfer deposit creation procedure for the legacy `Billing.Payment` system. When a customer sent a bank wire deposit in the pre-2011 era, customer service recorded the transfer using this procedure, linking the bank's wire reference number to the internal payment record.

The key element of a wire transfer is the `@TransactionID` (VARCHAR(20)) - the bank's own wire transfer reference number (often called SWIFT reference or bank transaction reference). This is stored in `dbo.WireTransferToPayment.TransactionID` for reconciliation with the bank. This is distinct from the internal 6-character `@PaymentTransactionID` (CHAR(6)) generated by `Billing.GetTransactionID()` for intra-system correlation.

Notable: `WireTransferToPayment` is referenced without a schema prefix, meaning it belongs to the `dbo` schema - the only such table among the legacy payment-type linking tables (all others use `Billing.*`).

Like all `Billing.Payment` procedures, this is legacy infrastructure frozen since January 2011.

## 2. Business Logic

### 2.1 Three-Table Atomic Transaction (Wire Transfer Variant)

**What**: Creates payment record, stores bank wire reference, logs status history atomically.

**Parameters Involved**: All parameters

**Rules**:
- @PaymentDate overwritten by GETUTCDATE()
- FundingTypeID=2 hardcoded (Wire Transfer)
- Internal @PaymentTransactionID CHAR(6) generated by Billing.GetTransactionID() (per-CID unique)
- INSERT Billing.Payment -> @PaymentID = SCOPE_IDENTITY()
- INSERT dbo.WireTransferToPayment (PaymentID, TransactionID=@TransactionID) - stores the bank reference
- INSERT History.Payment (status snapshot)
- ROLLBACK + return raw @@ERROR on Payment INSERT failure
- History error also triggers ROLLBACK
- SELECT @PaymentTransactionID + COMMIT + RETURN 0 on success

### 2.2 Dual TransactionID - Internal vs. Bank

| TransactionID | Type | Source | Stored In | Purpose |
|---|---|---|---|---|
| @PaymentTransactionID | CHAR(6) | Billing.GetTransactionID() | Billing.Payment.TransactionID | Internal correlation |
| @TransactionID | VARCHAR(20) | Bank wire reference | dbo.WireTransferToPayment.TransactionID | Bank reconciliation |

### 2.3 FundingType Map for All Legacy Payment Variants

| FundingTypeID | Payment Method | Procedure |
|---|---|---|
| 1 | Credit Card | PaymentByCreditCardAdd |
| 2 | Wire Transfer | PaymentByWireTransferAdd (this) |
| 3 | PayPal | PaymentByPayPalAdd |
| 5 | Western Union | PaymentByWesternUnionAdd |
| - | Neteller | PaymentByNetellerAdd (caller-supplied) |

## 8. Sample Queries

### 8.1 Find a legacy wire transfer payment by bank reference

```sql
SELECT
    bp.PaymentID,
    bp.CID,
    bp.Amount,
    bp.PaymentDate,
    bp.TransactionID AS InternalTXID,
    wtp.TransactionID AS BankWireReference
FROM Billing.Payment bp WITH (NOLOCK)
INNER JOIN dbo.WireTransferToPayment wtp WITH (NOLOCK) ON wtp.PaymentID = bp.PaymentID
WHERE wtp.TransactionID = 'SWIFT-REF-123';
```

### 8.2 Find all wire transfer payments for a customer

```sql
SELECT
    bp.PaymentID,
    bp.Amount,
    bp.PaymentDate,
    bp.TransactionID AS InternalTXID,
    wtp.TransactionID AS BankRef,
    ps.Name AS Status
FROM Billing.Payment bp WITH (NOLOCK)
INNER JOIN dbo.WireTransferToPayment wtp WITH (NOLOCK) ON wtp.PaymentID = bp.PaymentID
INNER JOIN Dictionary.PaymentStatus ps WITH (NOLOCK) ON ps.PaymentStatusID = bp.PaymentStatusID
WHERE bp.CID = 123456
ORDER BY bp.PaymentDate DESC;
```

### 8.3 Count legacy wire transfer payments by status

```sql
SELECT
    ps.Name AS Status,
    COUNT(*) AS Count,
    SUM(bp.Amount) AS TotalAmount
FROM Billing.Payment bp WITH (NOLOCK)
INNER JOIN dbo.WireTransferToPayment wtp WITH (NOLOCK) ON wtp.PaymentID = bp.PaymentID
INNER JOIN Dictionary.PaymentStatus ps WITH (NOLOCK) ON ps.PaymentStatusID = bp.PaymentStatusID
GROUP BY ps.Name
ORDER BY Count DESC;
```
