# Billing.PaymentByNetellerEdit

> Updates a legacy Neteller payment record with the gateway's confirmed response data - exchange rate, fee details, cardholder identity, and the Neteller-issued transaction reference - in a single atomic transaction.

## 1. Business Meaning

`Billing.PaymentByNetellerEdit` is the gateway-response update procedure for legacy Neteller payments. After a Neteller deposit is initiated via `Billing.PaymentByNetellerAdd`, the Neteller gateway processes the transaction and returns a callback with confirmed details. This procedure records that callback: the confirmed exchange rate, processing fees, the Neteller-generated transaction ID, and the cardholder's identity details (email, name).

The procedure serves as the second step in the two-phase Neteller payment flow: Add (create pending record) -> Edit (update with confirmed gateway response). The Neteller-provided TransactionID stored here is distinct from the internal 6-character TransactionID in `Billing.Payment` - the internal ID is for intra-system correlation, while the gateway TransactionID (VARCHAR(50)) is the external reference for Neteller reconciliation.

Like all `Billing.Payment` procedures, this is legacy infrastructure from the pre-2011 payment system.

## 2. Business Logic

### 2.1 Three-Table Atomic Update (Gateway Callback)

**What**: Atomically applies all gateway-response data across three related tables.

**Parameters Involved**: All parameters

**Rules**:
- Pre-check: Lookup @NetellerID by @AccountID - if not found RAISERROR(60025, "Cannot found neteller with specified properties") and return without starting transaction
- Step 1: UPDATE Billing.Payment SET ExchangeRate, TotalFee, DirectAcceptFee - if zero rows updated (PaymentID not found) -> ROLLBACK + RAISERROR(60000)
- Step 2: UPDATE Billing.Neteller SET SecureID - refreshes the stored Neteller security code
- Step 3: UPDATE Billing.NetellerToPayment SET Email, TransactionID, FirstName, LastName
- COMMIT on success / ROLLBACK on any error

### 2.2 Dual TransactionID Distinction

**What**: Two different TransactionIDs coexist in the Neteller payment flow.

**Parameters Involved**: @TransactionID, Billing.Payment.TransactionID

**Rules**:
- `Billing.Payment.TransactionID` (CHAR(6)): internal per-CID unique reference, generated by `Billing.GetTransactionID()` at deposit creation
- `@TransactionID` (VARCHAR(50)) in this procedure: the Neteller gateway's own transaction identifier, stored in `Billing.NetellerToPayment.TransactionID`
- These are different fields in different tables - the Neteller TransactionID is for external reconciliation with Neteller's records

### 2.3 Error Codes

| Code | Meaning | Trigger |
|------|---------|---------|
| 60025 | Neteller account not found by AccountID | Raised before transaction begins if AccountID not in Billing.Neteller |
| 60000 | General transaction failure | Any UPDATE error or PaymentID not found (@@ROWCOUNT=0) |

## 8. Sample Queries

### 8.1 Check Neteller payment with gateway response data

```sql
SELECT
    bp.PaymentID,
    bp.CID,
    bp.Amount,
    bp.ExchangeRate,
    bp.TotalFee,
    bp.DirectAcceptFee,
    ntp.Email,
    ntp.TransactionID AS NetellerTransactionID,
    ntp.FirstName,
    ntp.LastName
FROM Billing.Payment bp WITH (NOLOCK)
INNER JOIN Billing.NetellerToPayment ntp WITH (NOLOCK) ON ntp.PaymentID = bp.PaymentID
WHERE bp.PaymentID = 9876;
```

### 8.2 Find Neteller payments missing gateway callback data (TransactionID is NULL)

```sql
SELECT
    bp.PaymentID,
    bp.CID,
    bp.PaymentDate,
    n.AccountID
FROM Billing.Payment bp WITH (NOLOCK)
INNER JOIN Billing.NetellerToPayment ntp WITH (NOLOCK) ON ntp.PaymentID = bp.PaymentID
INNER JOIN Billing.Neteller n WITH (NOLOCK) ON n.NetellerID = ntp.NetellerID
WHERE ntp.TransactionID IS NULL
ORDER BY bp.PaymentDate DESC;
```

### 8.3 Reconcile Neteller payments by gateway TransactionID

```sql
SELECT
    ntp.TransactionID AS NetellerGatewayTXID,
    bp.TransactionID AS InternalTXID,
    bp.PaymentID,
    bp.Amount,
    bp.CID
FROM Billing.NetellerToPayment ntp WITH (NOLOCK)
INNER JOIN Billing.Payment bp WITH (NOLOCK) ON bp.PaymentID = ntp.PaymentID
WHERE ntp.TransactionID IS NOT NULL
ORDER BY bp.PaymentDate DESC;
```
