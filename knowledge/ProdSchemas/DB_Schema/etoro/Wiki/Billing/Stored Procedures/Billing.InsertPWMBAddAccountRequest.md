# Billing.InsertPWMBAddAccountRequest

> Inserts a new PayWithMyBank (Open Banking) account link request record into Billing.PWMBAddAccountRequest and simultaneously writes the same row to History.PWMBAddAccountRequest via the OUTPUT INTO clause for a complete audit trail.

## 1. Business Meaning

`Billing.InsertPWMBAddAccountRequest` records the initiation of a customer's bank account linking request through the PayWithMyBank (PWMB) Open Banking provider. PayWithMyBank is a UK Open Banking solution that lets customers pay directly from their bank account without a card. When the PWMB provider generates a transaction reference for a customer's account link attempt, this procedure stores that external reference alongside the customer ID and the initial status, creating the first row in both the live table and the history table simultaneously.

Without this procedure, eToro would have no record of initiated Open Banking link requests, making it impossible to track their resolution (linked, failed, or timed out) or support customer inquiries about the status of their bank account linking. The table currently holds 4,231 records from Jan-Oct 2023 with the majority reaching terminal states (Status=6 at 85%, Status=7 at 14%).

Data flows: the application calls this procedure when a PWMB provider generates an ExternalTransactionID for a new account-link request. At that point, FundingID is NULL because the Billing.Funding record for the bank account has not yet been created. The `OUTPUT ... INTO History.PWMBAddAccountRequest` clause ensures the initial state is written to audit history without a separate INSERT or trigger, matching the pattern used by UpdatePWMBAddAccountRequest for state transitions.

## 2. Business Logic

### 2.1 Dual-Write Pattern via OUTPUT INTO

**What**: The INSERT uses the `OUTPUT INSERTED.* INTO History.PWMBAddAccountRequest` pattern to simultaneously populate both the live table and the audit history table in a single statement.

**Columns/Parameters Involved**: `ExternalTransactionID`, `CID`, `FundingID`, `Status`, `InsertedTime`, `LastModificationTime`

**Rules**:
- Both `InsertedTime` and `LastModificationTime` are set to `GETUTCDATE()` at insert time - they are identical on creation
- `FundingID` is NULL on initial insert - the bank account (Billing.Funding) does not exist yet
- The OUTPUT clause writes the exact same values (via INSERTED.*) to History.PWMBAddAccountRequest - no separate INSERT is needed
- History row is created atomically with the live row - no window where the live row exists without a history record
- No IDENTITY column: `ExternalTransactionID` is the provider-generated natural key (VARCHAR(15)), so no SCOPE_IDENTITY() output is needed

**Diagram**:
```
Application -> InsertPWMBAddAccountRequest(@ExternalTransactionID, @CID, NULL, @Status)
                    |
                    v
          INSERT INTO Billing.PWMBAddAccountRequest
          (CID, FundingID, Status, ExternalTransactionID, InsertedTime, LastModificationTime)
          VALUES (@CID, NULL, @Status, @ExternalTransactionID, GETUTCDATE(), GETUTCDATE())
                    |
          OUTPUT INSERTED.* INTO History.PWMBAddAccountRequest
                    |
          +----------------------------------+
          |                                  |
    Billing.PWMBAddAccountRequest     History.PWMBAddAccountRequest
    (live record - ExternalTransID=PK) (audit row - identical values)
```

## 8. Sample Queries

### 8.1 Insert a new PWMB account link request (initial state)
```sql
EXEC Billing.InsertPWMBAddAccountRequest
    @ExternalTransactionID = 'PWMB2024001',
    @CID                   = 1234567,
    @FundingID             = NULL,
    @Status                = 1   -- pending/initiated
```

### 8.2 Check if a PWMB request was recorded and its current state
```sql
SELECT ExternalTransactionID, CID, FundingID, Status, InsertedTime, LastModificationTime
FROM Billing.PWMBAddAccountRequest WITH (NOLOCK)
WHERE ExternalTransactionID = 'PWMB2024001'
```

### 8.3 View full lifecycle of a PWMB request from history
```sql
SELECT ExternalTransactionID, CID, FundingID, Status, InsertedTime, LastModificationTime
FROM History.PWMBAddAccountRequest WITH (NOLOCK)
WHERE ExternalTransactionID = 'PWMB2024001'
ORDER BY LastModificationTime ASC
-- Shows each state transition from INSERT through final UpdatePWMBAddAccountRequest calls
```
