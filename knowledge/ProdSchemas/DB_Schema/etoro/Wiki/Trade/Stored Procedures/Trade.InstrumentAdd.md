# Trade.InstrumentAdd

> Creates a new tradeable instrument in Trade.Instrument by allocating the next InstrumentID from the internal ID generator and returning both the ID and the row's concurrency token (Passport).

## 1. Business Meaning

Trade.InstrumentAdd is the foundational write endpoint for instrument creation on the eToro platform. Every tradeable instrument - forex pairs, stocks, ETFs, crypto assets, commodities, indices - must first be registered in Trade.Instrument before positions can be opened, orders placed, or hedge contracts created. This procedure is the only correct way to create new instrument rows because it centralizes ID allocation through Internal.GetInstrumentID, ensuring uniqueness across replicated database instances.

Without this procedure, creating instruments would require callers to manage InstrumentID allocation themselves, risking collisions in replicated environments. The @Passport OUTPUT (rowversion/TIMESTAMP) is a concurrency token that allows the caller to detect concurrent modifications to the same instrument row - used for optimistic concurrency control in downstream operations.

Data flow: Instrument onboarding pipelines (including Trade.ProviderToInstrumentAdd which calls this procedure) invoke InstrumentAdd when a new currency pair or asset needs to be registered. The procedure allocates an ID, inserts the minimum required fields (buy/sell currency pair, trade range, dollar ratio), and returns the auto-generated rowversion. Additional instrument metadata (name, display properties, fees, etc.) is typically set via separate update procedures after creation. System versioning records the row to History.Instrument; audit triggers log to History.AuditHistory.

## 2. Business Logic

### 2.1 ID Allocation and Rowversion Return

**What**: InstrumentID is generated by an internal allocator; the Passport rowversion is captured via OUTPUT clause on the INSERT.

**Columns/Parameters Involved**: `@InstrumentID` (OUTPUT), `@Passport` (OUTPUT)

**Rules**:
- Internal.GetInstrumentID is called first to set @InstrumentID OUTPUT - this ensures IDs are unique even in multi-instance environments.
- The INSERT uses the OUTPUT..INTO pattern to capture INSERTED.Passport into a table variable, which is then SELECT'd into @Passport.
- TIMESTAMP/ROWVERSION is auto-generated by SQL Server on INSERT; it cannot be supplied by the caller.
- @Passport allows the caller to detect if the instrument row was modified between the INSERT and any subsequent read.

**Diagram**:
```
Caller provides: @BuyCurrencyID, @SellCurrencyID, @TradeRange, @DollarRatio
    |
    v
Internal.GetInstrumentID -> @InstrumentID OUTPUT
    |
    v
INSERT Trade.Instrument (InstrumentID, BuyCurrencyID, SellCurrencyID, TradeRange, DollarRatio)
    OUTPUT INSERTED.Passport INTO @Output
    |
    v
SELECT @Passport = Passport FROM @Output
    |
    v
RETURN 0
Outputs: @InstrumentID (new ID), @Passport (rowversion for concurrency)
```

### 2.2 Minimal Instrument Creation

**What**: The procedure inserts only the core pairing fields. Full instrument metadata is set later.

**Columns/Parameters Involved**: `@BuyCurrencyID`, `@SellCurrencyID`, `@TradeRange`, `@DollarRatio`

**Rules**:
- BuyCurrencyID + SellCurrencyID must form a unique pair (TISR_PAIR unique index on Trade.Instrument enforces this).
- For forex: BuyCurrencyID is the base currency, SellCurrencyID is the quote currency.
- For stocks/crypto: BuyCurrencyID is the asset's Dictionary.Currency ID (same as InstrumentID for major assets), SellCurrencyID is the denomination currency.
- DollarRatio: price normalization multiplier. Most instruments use 1; JPY pairs use 100.
- TradeRange: the spread/range setting for order matching. SMALLINT.

## 8. Sample Queries

### 8.1 Add a new forex instrument (EUR/GBP)

```sql
DECLARE @NewID INT, @NewPassport TIMESTAMP;
EXEC Trade.InstrumentAdd
    @InstrumentID   = @NewID OUTPUT,
    @BuyCurrencyID  = 2,     -- EUR
    @SellCurrencyID = 3,     -- GBP
    @TradeRange     = 10,
    @DollarRatio    = 1.0,
    @Passport       = @NewPassport OUTPUT;

SELECT @NewID AS NewInstrumentID, @NewPassport AS Passport;
```

### 8.2 Verify the inserted instrument

```sql
SELECT InstrumentID, BuyCurrencyID, SellCurrencyID, TradeRange, DollarRatio
FROM Trade.Instrument WITH (NOLOCK)
WHERE InstrumentID = @NewID;
```

### 8.3 View instrument with currency names

```sql
SELECT
    i.InstrumentID,
    buy.Abbreviation AS BuyCurrency,
    sell.Abbreviation AS SellCurrency,
    i.TradeRange,
    i.DollarRatio
FROM Trade.Instrument i WITH (NOLOCK)
JOIN Dictionary.Currency buy WITH (NOLOCK) ON buy.CurrencyID = i.BuyCurrencyID
JOIN Dictionary.Currency sell WITH (NOLOCK) ON sell.CurrencyID = i.SellCurrencyID
WHERE i.InstrumentID = @NewID;
```
