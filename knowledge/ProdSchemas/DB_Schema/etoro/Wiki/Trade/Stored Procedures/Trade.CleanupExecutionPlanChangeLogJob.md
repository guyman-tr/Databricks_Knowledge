# Trade.CleanupExecutionPlanChangeLogJob

> Archives completed execution plan change log entries from Trade.ExecutionPlanChangeLog to History.ExecutionPlanChangeLog using MERGE, then deletes the archived rows. Part of the US CleanupJob.

## 1. Business Meaning

Trade.CleanupExecutionPlanChangeLogJob moves completed execution plan change log entries from the hot operational table to history. A change log entry is considered "completed" when its OrderID no longer appears in either Trade.OrderForClose or Trade.OrderForOpen (both open and close order queues).

This procedure captures the audit trail of changes to execution plans during order processing: unit adjustments, level changes, amount modifications, correlation ID updates, position assignments, and settlement type changes. These records are important for troubleshooting order execution issues.

Part of the `[etoro - US CleanupJob]` SQL Agent job.

---

## 2. Business Logic

### 2.1 Completed Order Detection

**What**: Finds OrderIDs in Trade.ExecutionPlanChangeLog that are not in Trade.OrderForClose AND not in Trade.OrderForOpen.

**Rules**:
- Uses EXCEPT with UNION ALL of both order queues (close + open)
- This is the only cleanup procedure that checks BOTH order queues, because change logs can be generated by both open and close operations

### 2.2 MERGE to History

**What**: Upserts into History.ExecutionPlanChangeLog with partition elimination on OccurredAsDate (last 30 days).

**Columns Archived**: ChangeLogID, ChangeOccurred, OrderID, Units, Level, Amount, OpenCorrelationID, PositionID, SettlementType

**Match Key**: ChangeLogID (unique per change log entry)

### 2.3 Delete from Active

**What**: Removes archived rows from Trade.ExecutionPlanChangeLog after successful MERGE.

**Rules**:
- JOIN delete on ChangeLogID (not OrderID, since one order can have multiple change log entries)
