## 1. Business Meaning

Trade.PositionOpenRequestAdd is the entry-point for enqueuing a new position open request in the trading system. When a client (user or system) places a trade, this SP is called to: (1) generate a new unique PositionID via Internal.GetPositionID_Bigint, and (2) insert a row into Trade.PositionRequest with RequestType=1 so that the downstream execution engine can process it.

The SP performs three pre-flight validations before inserting:
- **Amount > 0**: the position amount must be positive (error 60074)
- **AmountInUnitsDecimal > 0**: the units must be positive (error 60075)
- **OrderID uniqueness**: if @OrderID is provided, it must not already exist in Trade.Position or History.Position (error 60031) - prevents duplicate-order execution

An additional validation is present for the ForexResultID from @AdditionalParam (via Internal.ParseTradeAdditionalParam), though the actual ForexResult existence check is commented out - it was disabled at some point (Game.ForexResult check).

The SP is the symmetric counterpart to Trade.PositionCloseRequestAdd: together they handle the full request lifecycle.

## 2. Business Logic

### 2.1 Amount and Units Validation

**What**: Validates that Amount and AmountInUnitsDecimal are positive before proceeding.

**Columns/Parameters Involved**: @Amount, @AmountInUnitsDecimal

**Rules**:
- IF @Amount <= 0: RAISERROR(60074, 16, 1); RETURN 60074
- IF @AmountInUnitsDecimal <= 0: RAISERROR(60075, 16, 1); RETURN 60075
- Error 60074 = "Amount is not positive" for open request
- Error 60075 = "AmountInUnitsDecimal is not positive" for open request

### 2.2 AdditionalParam Parsing (ForexResultID)

**What**: Parses @AdditionalParam to extract ForexResultID via Internal.ParseTradeAdditionalParam.

**Columns/Parameters Involved**: @AdditionalParam, @ForexResultID, Internal.ParseTradeAdditionalParam

**Rules**:
- @AdditionalParam is SQL_VARIANT; the parsing function extracts trading metadata including ForexResultID
- If parsing errors: RAISERROR(60000); RETURN 60000
- The ForexResultID existence check against Game.ForexResult is commented out - was formerly validated but disabled

### 2.3 OrderID Duplicate Check

**What**: If @OrderID is provided, ensures it doesn't already exist in Trade.Position or History.Position.

**Columns/Parameters Involved**: @OrderID, Trade.Position.OrderID, History.Position.OrderID

**Rules**:
- Only checked if ISNULL(@OrderID, 0) <> 0 (i.e., OrderID is not NULL and not 0)
- Checks both Trade.Position (open positions) and History.Position (closed positions)
- If found in either: error 60031 "OrderID already exists"; RETURN 60031
- Prevents double-execution of the same order

### 2.4 PositionID Generation and Request Insert

**What**: Generates a unique PositionID via the internal ID generator and inserts the open request into Trade.PositionRequest.

**Columns/Parameters Involved**: @PositionID OUTPUT, Internal.GetPositionID_Bigint, Trade.PositionRequest.RequestType=1

**Rules**:
- @PositionID is initialized to NULL before the generate call
- EXECUTE Internal.GetPositionID_Bigint @PositionID OUTPUT: generates a globally unique BIGINT for this position
- INSERT INTO Trade.PositionRequest with RequestType=1 (open request)
- All position parameters inserted: CID, CurrencyID, ProviderID, GameServerID, InstrumentID, OrderID, Leverage, Amount, AmountInUnitsDecimal, UnitMargin, LotCountDecimal, IsBuy, CloseOnEndOfWeek, AdditionalParam, TradeRange, ParentPositionID
- ParentPositionID=0 when not a copy-trade mirror position
- Returns @@ERROR (0 on success, non-zero on failure)

```sql
SELECT TOP 20 PositionID, CID, InstrumentID, Amount, AmountInUnitsDecimal, Occurred
```
