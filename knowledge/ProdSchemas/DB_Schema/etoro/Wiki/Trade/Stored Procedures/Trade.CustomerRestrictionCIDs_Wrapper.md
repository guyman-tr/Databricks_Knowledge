# Trade.CustomerRestrictionCIDs_Wrapper

> Synchronizes trading restrictions for a batch of CIDs: sets restrictions for the provided list via Trade.CustomerRestrictionSet_CIDs, then removes restrictions for customers NOT in the list (but currently blocked) via Trade.CustomerRestrictionRemove_CIDs.

## 1. Business Meaning

Trade.CustomerRestrictionCIDs_Wrapper is a synchronization procedure that ensures the set of restricted customers matches the provided CID list exactly. It operates in two steps:

1. **Set restrictions**: Calls Trade.CustomerRestrictionSet_CIDs for all CIDs in the input list, creating new block records for those not already blocked.
2. **Remove restrictions**: Identifies customers who ARE currently blocked (for the same OperationTypeID and BlockReasonID) but are NOT in the input list, then calls Trade.CustomerRestrictionRemove_CIDs to unblock them.

This "sync" pattern ensures that the active set of blocked customers precisely reflects the current risk assessment — customers whose risk score has dropped below the threshold are automatically unblocked, while new high-risk customers are blocked.

## 2. Business Logic

### 2.1 Removal Set Calculation

**What**: Identifies customers to unblock by finding currently-blocked CIDs absent from the input list.

**Rules**:
- LEFT JOIN: Customer.BlockedCustomerOperations LEFT JOIN @CID → WHERE c.CID IS NULL
- Filtered by same OperationTypeID and BlockReasonID
- These are customers whose restriction is no longer warranted

### 2.2 Dual-Operation Sync

**What**: Atomically applies the desired restriction state.

**Rules**:
- Trade.CustomerRestrictionSet_CIDs handles idempotent insertion (skips existing blocks)
- Trade.CustomerRestrictionRemove_CIDs handles atomic removal + archival
- Order matters: SET first (to avoid briefly removing then re-adding the same customer)

## 8. Sample Queries

### 8.1 Preview which customers would be unblocked

```sql
DECLARE @CID Trade.CidList;
-- populate @CID with the desired list...
SELECT  a.CID
FROM    Customer.BlockedCustomerOperations a WITH (NOLOCK)
LEFT JOIN @CID c ON a.CID = c.CID
WHERE   c.CID IS NULL
        AND a.OperationTypeID = 2
        AND a.BlockReasonID = 2;
```
