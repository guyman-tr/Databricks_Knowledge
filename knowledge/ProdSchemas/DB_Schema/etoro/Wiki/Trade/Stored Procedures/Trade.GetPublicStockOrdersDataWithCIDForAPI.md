> Returns a public-safe subset of pending stock orders from Stocks.Orders for a given customer, covering both entry (open) and exit (close) order types.

## 1. Business Meaning

This procedure retrieves all pending stock orders for a customer from the `Stocks.Orders` table (cross-schema). Stock orders in eToro represent the order-routing layer for real stock positions - they capture open requests that are pending execution against a market counterparty. Unlike Trade.Orders (CFD/trade engine orders), Stocks.Orders handles orders routed through the stock settlement infrastructure.

The procedure lives in the Trade schema but reads from Stocks.Orders - it is a cross-schema public API read that sits alongside `Trade.GetPublicOrdersDataWithCIDForAPI` (for CFD orders) as the stock-specific equivalent. The "Public" naming convention means this feeds public-facing API endpoints and returns only a reduced, non-sensitive subset of Stocks.Orders columns (8 of 30+ columns in the full table). Fields such as Amount, Leverage, PriceOnRequest, and HedgeID are excluded.

Data flows: SELECT all orders for @cid from Stocks.Orders WITH NOLOCK. Both entry orders (IsEntry=1, position-opening) and exit orders (IsEntry=0, position-closing) are returned in a single result set, differentiated by the IsEntry column.

## 2. Business Logic

### 2.1 Entry vs Exit Orders via IsEntry

**What**: A single query returns both open-position requests and close-position requests for the customer, distinguished by IsEntry.

**Columns/Parameters Involved**: `IsEntry`, `PositionID`

**Rules**:
- IsEntry=1: This order requests opening a new stock position. PositionID is NULL until the order executes.
- IsEntry=0: This order requests closing an existing stock position. PositionID references the position to close.
- ISNULL(PositionID, 0) coerces unexecuted entry orders to 0.

**Diagram**:
```
Stocks.Orders WHERE CID = @cid
  |
  +-- IsEntry=1 (Entry orders - opening new positions)
  |     PositionID = 0 (not yet assigned)
  |
  +-- IsEntry=0 (Exit orders - closing existing positions)
        PositionID = linked position being closed
```

### 2.2 Copy-Trade Order Identification via MirrorID

**What**: Identifies whether an order was initiated by a copy-trade mirror relationship.

**Columns/Parameters Involved**: `MirrorID`

**Rules**:
- MirrorID=0 (after ISNULL coercion): manual order placed directly by the customer.
- MirrorID>0: order was auto-generated by a copy-trade mirror (customer is copying a leader).

## 8. Sample Queries

### 8.1 Get all pending stock orders for a customer

```sql
EXEC Trade.GetPublicStockOrdersDataWithCIDForAPI @cid = 14952810;
```

### 8.2 Separate entry and exit stock orders for a customer

```sql
SELECT so.OrderID, so.CID, so.InstrumentID, so.IsBuy, so.IsEntry,
       ISNULL(so.MirrorID, 0) AS MirrorID, so.OpenRequest,
       ISNULL(so.PositionID, 0) AS PositionID
FROM Stocks.Orders so WITH (NOLOCK)
WHERE so.CID = 14952810
ORDER BY so.IsEntry DESC, so.OpenRequest DESC;
-- IsEntry=1 (entry orders) first, then IsEntry=0 (exit orders)
```

### 8.3 Count pending stock orders by type for a customer

```sql
SELECT
    CASE WHEN IsEntry = 1 THEN 'Entry (Open)' ELSE 'Exit (Close)' END AS OrderType,
    COUNT(*) AS PendingCount
FROM Stocks.Orders WITH (NOLOCK)
WHERE CID = 14952810
GROUP BY IsEntry;
```
