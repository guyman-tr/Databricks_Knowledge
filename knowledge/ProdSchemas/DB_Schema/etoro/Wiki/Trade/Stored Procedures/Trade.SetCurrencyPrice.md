## 1. Business Meaning

`Trade.CurrencyPrice` holds the **current live market price** for each instrument+provider pair - the single most frequently updated table in the trading system. This procedure is the write interface for updating a price tick.

When a price feed event arrives from a liquidity provider, this procedure records:
- **Bid/Ask**: The new market prices
- **Occurred**: Server-side UTC timestamp of when the price was processed
- **OccurredOnServer**: The timestamp reported by the price feed source
- **ReceivedOnPriceServer**: When the price server received the tick (optional)
- **PriceRateID**: Sequential ID linking to the price history (from a sequence/identity source)
- **MarketPriceRateID**: Optional reference to the market price record
- **LastPrice**: The previous price (or 0 if not provided)

The procedure is a simple UPDATE with no INSERT fallback - it assumes the CurrencyPrice row for the instrument+provider already exists (pre-seeded). If no row matches, the UPDATE silently affects 0 rows (no error raised).

The commented-out code shows that historically `Internal.GetPriceRateID` was called to generate the @PriceRateID - this was moved to the caller as of the refactor reflected in `Trade.SetCurrencyPriceHistoryInsert_New`.

## 2. Business Logic

### 2.1 Price Record Update

**What**: Overwrites the live price data for the instrument+provider row.

**Columns/Parameters Involved**: `Trade.CurrencyPrice` all updatable price columns

**Rules**:
- UPDATE WHERE ProviderID=@ProviderID AND InstrumentID=@InstrumentID
- Sets all 8 price/timestamp/ID columns atomically
- @Occurred is computed internally: `DECLARE @Occurred DATETIME = GETUTCDATE()` - caller cannot override this timestamp
- RETURN 0 on success
- CATCH: THROW (re-raises); the SET @LocalError after THROW is unreachable dead code

**Column mapping**:
| Column | Source |
|--------|--------|
| Bid | @Bid (dtPrice custom type) |
| Ask | @Ask (dtPrice custom type) |
| Occurred | GETUTCDATE() (server-side UTC, not from caller) |
| OccurredOnServer | @OccurredOnServer (feed source timestamp) |
| PriceRateID | @PriceRateID (generated by caller) |
| ReceivedOnPriceServer | @ReceivedOnPriceServer (nullable) |
| MarketPriceRateID | @MarketPriceRateID (nullable) |
| LastPrice | @LastPrice (default 0) |

## 8. Sample Queries

### 8.1 Update the live price for an instrument+provider

```sql
EXEC Trade.SetCurrencyPrice
    @InstrumentID = 1234,
    @ProviderID = 5,
    @Bid = 150.25,
    @Ask = 150.27,
    @OccurredOnServer = '2026-03-17 10:30:00',
    @PriceRateID = 9876543210,
    @ReceivedOnPriceServer = '2026-03-17 10:30:00.100',
    @MarketPriceRateID = NULL,
    @LastPrice = 150.20
```

### 8.2 Check current live prices for an instrument

```sql
SELECT cp.InstrumentID, cp.ProviderID, cp.Bid, cp.Ask,
    cp.Occurred, cp.OccurredOnServer, cp.PriceRateID, cp.LastPrice
FROM Trade.CurrencyPrice cp WITH (NOLOCK)
WHERE cp.InstrumentID = 1234
ORDER BY cp.ProviderID
```
