## 1. Business Meaning

Trade.InsertTradonomyContract is the write endpoint for creating new Tradonomi contracts - the records that link eToro instruments to external liquidity provider (LP) contract periods. Each contract defines a validity window (FromDate to ToDate) for an instrument, a description (typically the instrument symbol such as EURUSD), and an active flag. Contracts are the bridge between eToro's internal instrument IDs and the LP contract identifiers used for hedging and order routing.

Without this procedure there would be no consistent way to create new contract entries. Parallel to Trade.InsertTradonomiContactDailySchedule, this procedure follows the same pattern of delegating ID generation to an Internal schema procedure (Internal.GetTradonomiContractsID) before executing the INSERT. The caller receives the newly assigned @ContractID OUTPUT for immediate use - for example, to then call Trade.SetTradonomiToLPContracts to link the new contract to LP contracts.

Data flow: Configuration or instrument onboarding tools call this procedure when a new instrument or contract period is set up. The ContractID is allocated, the contract row is inserted into Trade.TradonomiContracts, and the @ContractID is returned. Trade.MarkTradonomiContractAsActive can subsequently activate the contract (set IsActive=1). History is maintained by system versioning (History.TradonomiContracts).

## 2. Business Logic

### 2.1 ID Generation Before Insert with Dual Error Signaling

**What**: ContractID is generated by an internal ID generator before the INSERT. On failure, the caller receives both a return code and a sentinel output value.

**Columns/Parameters Involved**: `@ContractID` (OUTPUT), RETURN code

**Rules**:
- Internal.GetTradonomiContractsID is called first to set @ContractID OUTPUT.
- The @ContractID is then used in the VALUES clause of the INSERT - not auto-generated by the table.
- On any error in the TRY block, the CATCH block sets @ContractID = -1 AND returns -1 (RETURN (-1)).
- This dual signaling means callers can detect failure either by checking RETURN value OR by examining @ContractID after the call.
- On success, RETURN (0) and @ContractID has the valid new contract ID.

**Diagram**:
```
Inputs: @InstrumentID, @IsActive, @FromDate, @ToDate, @Description
    |
    v
TRY:
    EXEC Internal.GetTradonomiContractsID @ContractID OUTPUT
        -> @ContractID = next available contract ID
    |
    v
    INSERT Trade.TradonomiContracts
        (ContractID, InstrumentID, IsActive, FromDate, ToDate, Description)
    |
    v
    RETURN 0   [success - @ContractID is valid]

CATCH:
    SET @ContractID = -1
    RETURN -1  [failure - caller can check either indicator]

Output: @ContractID (valid ID on success, -1 on failure)
```

### 2.2 IsActive on Insert

**What**: The procedure allows setting IsActive directly on creation, but only one contract per instrument should be active.

**Columns/Parameters Involved**: `@IsActive`, `@InstrumentID`

**Rules**:
- @IsActive (TINYINT): 0 = inactive (historical or pending), 1 = active (currently used for routing and LP mapping).
- The procedure does NOT enforce the one-active-per-instrument rule at INSERT time - it simply writes whatever @IsActive value is passed.
- Trade.MarkTradonomiContractAsActive is the correct mechanism to set one contract active and deactivate others atomically.
- Inserting with @IsActive=1 without first calling MarkTradonomiContractAsActive may leave multiple active contracts for the same instrument.

## 8. Sample Queries

### 8.1 Create a new inactive contract for an instrument

```sql
DECLARE @NewContractID INT;
EXEC Trade.InsertTradonomyContract
    @InstrumentID = 1,
    @IsActive     = 0,
    @FromDate     = '2026-01-01',
    @ToDate       = '2099-12-31',
    @Description  = 'EURUSD',
    @ContractID   = @NewContractID OUTPUT;

SELECT @NewContractID AS AssignedContractID;
-- Returns the new ContractID (or -1 on failure)
```

### 8.2 Check the RETURN code and output together

```sql
DECLARE @NewContractID INT;
DECLARE @RC INT;

EXEC @RC = Trade.InsertTradonomyContract
    @InstrumentID = 17,
    @IsActive     = 0,
    @FromDate     = '2026-01-01',
    @ToDate       = '2099-12-31',
    @Description  = 'NewInstrument',
    @ContractID   = @NewContractID OUTPUT;

IF @RC = 0
    SELECT @NewContractID AS NewContractID, 'SUCCESS' AS Status;
ELSE
    SELECT @NewContractID AS NewContractID, 'FAILED' AS Status;
-- @NewContractID will be -1 if @RC = -1
```

### 8.3 Verify inserted contract

```sql
SELECT ContractID, InstrumentID, IsActive, FromDate, ToDate, Description
FROM Trade.TradonomiContracts WITH (NOLOCK)
WHERE InstrumentID = 1
ORDER BY ContractID DESC;
```
