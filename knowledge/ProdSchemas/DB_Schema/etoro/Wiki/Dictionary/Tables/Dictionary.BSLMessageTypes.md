## 1. Business Meaning

Dictionary.BSLMessageTypes defines the types of messages generated by the BSL (Balance Stop-Loss) system, which monitors customer equity levels and triggers protective actions when equity drops below configured thresholds. The BSL system is the platform's margin call mechanism — it warns customers when their equity is dangerously low, forces liquidation of positions when equity falls below the minimum threshold, and unblocks accounts when equity recovers.

Without this table, the BSL message queue system (used by `Trade.InsertBSLMessagesIntoQueue`) would have no way to classify whether a queued message represents a warning, a liquidation event, or an unblock event. The `Trade.GetUsersFromBSLTables` procedure JOINs to this table to resolve message type IDs into human-readable operation names for BSL event reporting.

The BSL system reads equity thresholds from `Dictionary.BSLOperationThreshold`, where each threshold row references a `MessageTypeID` from this table — linking the trigger percentage to the type of message that should be generated when that threshold is crossed.

## 2. Business Logic

### 2.1 BSL Equity Protection Lifecycle

**What**: Three-stage equity protection system triggered by declining customer equity.

**Columns/Parameters Involved**: `ID`, `MessageTypeDecstiption`

**Rules**:
- **Warning (ID=1)**: Generated when customer equity drops below alert thresholds (20% first alert, 10% second alert per BSLOperationThreshold). Customer receives a notification to add funds or close positions. No forced action taken.
- **Liquidation (ID=2)**: Generated when customer equity drops below the liquidation threshold (5% per BSLOperationThreshold). The system forcibly closes positions to prevent the account from going negative. This is a protective measure for both the customer and the platform.
- **Unblock (ID=3)**: Generated when customer equity recovers above the unblock threshold (25% per BSLOperationThreshold). Accounts that were restricted during low-equity conditions are automatically unblocked to resume normal trading.

**Diagram**:
```
Customer Equity Monitoring (BSL)

  100% ─────────────────────────── Normal trading
   25% ─── Unblock threshold ───── ID=3: Unblock user
   20% ─── First alert ─────────── ID=1: Warning about liquidation
   10% ─── Second alert ────────── ID=1: Warning about liquidation
    5% ─── Liquidation threshold ─ ID=2: Forced liquidation
    0% ─────────────────────────── Account equity depleted

Thresholds from Dictionary.BSLOperationThreshold
```

## 3. Data Overview

| ID | MessageTypeDecstiption | Meaning |
|---|---|---|
| 1 | Warning about liquidation | Sent when customer equity drops below alert thresholds (20% or 10%). Two-stage warning — first alert at 20%, second at 10%. Urges customer to deposit funds or close losing positions before forced liquidation. |
| 2 | Liquidation | Triggered when equity falls below 5%. System forcibly closes customer positions to prevent negative balance. Critical risk management event logged for compliance and audit. |
| 3 | Unblock user | Triggered when equity recovers above 25%. Restores normal trading operations on an account that was previously restricted due to dangerously low equity levels. |

## 8. Sample Queries

### 8.1 List all BSL message types
```sql
SELECT  ID,
        MessageTypeDecstiption
FROM    Dictionary.BSLMessageTypes WITH (NOLOCK)
ORDER BY ID;
```

### 8.2 Show BSL thresholds with their message types
```sql
SELECT  BOT.ID AS ThresholdID,
        BOT.Name AS ThresholdName,
        BOT.ValueInPercent,
        BMT.MessageTypeDecstiption AS MessageType
FROM    Dictionary.BSLOperationThreshold BOT WITH (NOLOCK)
INNER JOIN Dictionary.BSLMessageTypes BMT WITH (NOLOCK)
        ON BMT.ID = BOT.MessageTypeID
ORDER BY BOT.ValueInPercent DESC;
```

### 8.3 Query BSL events with resolved message type names
```sql
SELECT  BMT.MessageTypeDecstiption AS BSLOperation,
        BMT.ID AS MessageTypeID
FROM    Dictionary.BSLMessageTypes BMT WITH (NOLOCK)
ORDER BY BMT.ID;
```
