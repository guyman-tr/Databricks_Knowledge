## 1. Business Meaning

Dictionary.CEPNamedListTypeID classifies the types of named lists used in the CEP (Complex Event Processing) rules engine. Named lists are collections of values (customer IDs, instrument IDs, country codes, etc.) that CEP rules can reference when evaluating conditions — for example, "if customer is in the VIP list" or "if instrument is in the restricted list."

The two types distinguish how the list is populated: Normal lists (ID=1) are manually configured by operations teams through a UI, while DB Generated lists (ID=2) are automatically populated by database queries that run periodically. This distinction affects how the CEP engine refreshes list contents — manual lists change only when a human edits them, while DB-generated lists are auto-refreshed based on live data.

The `CEP.NamedLists` table has an explicit FK (`FK_NamedLists_CEPNamedListTypeID`) referencing this table, ensuring every named list in the CEP system has a valid type classification.

## 2. Business Logic

### 2.1 List Population Strategies

**What**: Two methods for populating CEP named lists.

**Columns/Parameters Involved**: `NamedListTypeID`, `Description`

**Rules**:
- **Normal (ID=1)**: List entries are manually maintained by operations or compliance teams through the CEP management UI. Content changes only when a human adds/removes entries. Used for curated lists like VIP customers, sanctioned entities, or manually flagged instruments.
- **DB Generated (ID=2)**: List entries are automatically generated by a database query that runs on a schedule. Content is refreshed periodically without human intervention. Used for dynamic lists like "all customers with equity > $100K" or "all instruments in a specific exchange."

### 8.2 Count named lists by type
```sql
SELECT  NLT.NamedListTypeID,
        NLT.Description,
        COUNT(NL.NamedListTypeID) AS ListCount
FROM    Dictionary.CEPNamedListTypeID NLT WITH (NOLOCK)
LEFT JOIN CEP.NamedLists NL WITH (NOLOCK)
        ON NL.NamedListTypeID = NLT.NamedListTypeID
GROUP BY NLT.NamedListTypeID, NLT.Description
ORDER BY NLT.NamedListTypeID;
```

### 8.3 Find all DB-generated named lists
```sql
SELECT  NL.*
FROM    CEP.NamedLists NL WITH (NOLOCK)
INNER JOIN Dictionary.CEPNamedListTypeID NLT WITH (NOLOCK)
        ON NLT.NamedListTypeID = NL.NamedListTypeID
WHERE   NLT.Description = 'DB Generated Named List';
```
