# Customer.RegisterDemo

> Creates a new demo (paper trading) customer account: validates username uniqueness, ensures affiliate exists, inserts Customer.Customer (IsReal=0), initializes CustomerMoney and BackOffice.Customer, seeds $100K demo balance via Billing.AmountAdd, and optionally sends a lead notification via Service Broker.

## 1. Business Meaning

`Customer.RegisterDemo` is the demo account creation endpoint. It provisions a full but non-real (IsReal=0) customer record, initializing all tables required for the customer to log in and trade in the paper trading environment. Demo accounts receive a $100,000 virtual balance seeded via `Billing.AmountAdd` (10,000,000 cents with deposit type 1).

Key characteristics of demo accounts created here:
- `IsReal = 0` in Customer.Customer - flagged as demo throughout the system.
- Password stored as empty string `''` - actual authentication is handled by the STS system, not the database.
- `AccountExpirationDate` is typically set to `9999-12-31` for perpetual demo accounts.
- GCID is passed in (pre-generated by UserDBAPI), not generated here.

The procedure also conditionally notifies the lead tracking system via Service Broker (`svcLead`) if `Maintenance.Feature.FeatureID=3` is active and the customer's original CID was zero (`@WasOrigCIDZero=1`), which indicates a fresh registration rather than a CID reuse.

## 2. Business Logic

### 2.1 Username Uniqueness Guard

**What**: Prevents duplicate username registration.

**Rules**:
- Checks `Customer.Customer.UserName_LOWER = LOWER(@UserName)`.
- If exists: COMMIT (release lock), RaisError with username-specific message, RETURN -1.

### 2.2 Affiliate Auto-Provisioning

**What**: Ensures the affiliate (SerialID) exists in BackOffice.Affiliate before the customer is created.

**Rules**:
- IF NOT EXISTS in BackOffice.Affiliate WHERE AffiliateID = @SerialID: INSERT with AffiliateStatusID=1, ManagerID=NULL.
- Prevents FK constraint violations when the customer's SerialID doesn't yet have an affiliate record.

### 2.3 Customer Record Creation

**What**: Inserts the full demo customer row across three tables.

**Rules**:
- INSERT `Customer.Customer`: 33 columns, IsReal=0, Password='', OriginalCID = CASE ISNULL(@OrigCID,0) WHEN 0 THEN @CID ELSE @OrigCID END, RegionID from `Internal.GetRegionIDByIP(@RegIP)`.
- INSERT `Customer.CustomerMoney`: CID, GCID, Credit=0.
- INSERT `BackOffice.Customer`: CID, CashoutFeeGroupID (from Trade.CashoutRange WHERE IsDefault=1), ChangePassword, RiskStatusID, AccountTypeID, RegulationID, RegulationChangeDate=@Occurred.

### 2.4 Demo Balance Seeding

**What**: Gives the demo customer $100,000 virtual balance.

**Rules**:
- EXEC `Billing.AmountAdd(@CID, @CurrencyID, 1, 10000000)` - deposit type 1, 10,000,000 cents = $100,000.
- If @Answer != 0: RAISERROR(60000).

### 2.5 Lead Notification (Conditional)

**What**: Notifies the lead tracking system about new self-registered customers.

**Rules**:
- Only if `@WasOrigCIDZero = 1` (customer had no prior OrigCID - fresh registration).
- Reads `Maintenance.Feature WHERE FeatureID=3` for lead mode flag.
- If lead mode = 1: sends XML via Service Broker (`svcLead`, contract `ctrAnyXMLData`).

```
Validate username uniqueness
Ensure BackOffice.Affiliate exists for @SerialID
BEGIN TX:
  INSERT Customer.Customer (IsReal=0)
  INSERT Customer.CustomerMoney (Credit=0)
  INSERT BackOffice.Customer
  EXEC Billing.AmountAdd (10,000,000 cents demo balance)
  IF @WasOrigCIDZero=1 AND FeatureID3=1: Service Broker lead notification
COMMIT
```

## 8. Sample Queries

### 8.1 Find recently created demo accounts

```sql
SELECT TOP 20
    c.CID,
    c.GCID,
    c.UserName,
    c.CountryID,
    c.Registered,
    c.PlayerLevelID,
    c.ReferralID
FROM Customer.Customer c WITH (NOLOCK)
WHERE c.IsReal = 0
ORDER BY c.Registered DESC
```

### 8.2 Check demo account balance

```sql
SELECT
    cm.CID,
    cm.GCID,
    cm.Credit,
    cm.Balance
FROM Customer.CustomerMoney cm WITH (NOLOCK)
WHERE cm.CID = 12345
```

### 8.3 Find affiliates without BackOffice.Affiliate records

```sql
SELECT DISTINCT c.SerialID
FROM Customer.Customer c WITH (NOLOCK)
WHERE c.SerialID IS NOT NULL
  AND NOT EXISTS (
    SELECT 1 FROM BackOffice.Affiliate ba WITH (NOLOCK)
    WHERE ba.AffiliateID = c.SerialID
  )
```
