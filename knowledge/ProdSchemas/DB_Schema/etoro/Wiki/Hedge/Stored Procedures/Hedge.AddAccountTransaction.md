## 1. Business Meaning

`Hedge.AddAccountTransaction` logs a single financial transaction that occurred on a liquidity provider account. It records what happened (the transaction type), how much (the amount), whether it was a manual operation, when it occurred at the account, and an optional comment. It optionally associates the transaction with a specific instrument.

The target table `dbo.HedgeAccountTransactions` (dbo schema, not Hedge schema) serves as the transaction ledger for LP accounts - tracking cash movements and adjustments to account balances over time. These transactions represent events like: cash deposits to the LP account, cash withdrawals, commissions charged by the LP, or manual adjustments made by eToro's hedge operations team.

The `@IsManual` flag distinguishes automated system-generated transactions from manual entries made by the hedge operations team for corrections or reconciliation. The `@Comment` field provides free-text justification for manual transactions.

Data flow: Called by hedge server components when account financial events are detected. No callers found in the SSDT project, indicating the caller is the external hedge server application.

Note: The target table `dbo.HedgeAccountTransactions` (without schema prefix implies dbo) is not tracked in the SSDT Hedge schema project, suggesting it may be a legacy or cross-schema table managed separately.

## 2. Business Logic

### 2.1 Manual vs Automated Transaction Distinction

**What**: The `@IsManual` flag marks whether this transaction was system-generated or manually entered.

**Columns/Parameters Involved**: `@IsManual`, `@Comment`

**Rules**:
- `@IsManual = 0`: Automated transaction from the hedge engine (e.g., LP charging commission)
- `@IsManual = 1`: Manual transaction entered by hedge ops team (reconciliation, adjustment)
- `@Comment` is optional (defaults to empty string '') but typically populated for manual transactions
- `@HedgeServerID` defaults to 0 when the transaction is account-level (not server-specific)
- `@InstrumentID` defaults to NULL for account-wide transactions not tied to a specific instrument

## 8. Sample Queries

### 8.1 Execute: Log a manual cash deposit to an LP account

```sql
EXEC Hedge.AddAccountTransaction
    @LiquidityAccountID  = 101,
    @TransactionTypeID   = 1,    -- deposit type
    @Amount              = 50000.00,
    @IsManual            = 1,
    @OccurredAtAccount   = '2026-03-19 09:00:00',
    @Comment             = 'Manual funding for hedge expansion Q1 2026',
    @HedgeServerID       = 0,
    @InstrumentID        = NULL
```

### 8.2 Query: Recent transactions for a liquidity account

```sql
SELECT TOP 20
    HedgeServerID,
    LiquidityAccountID,
    TransactionTypeID,
    Amount,
    IsManual,
    OccurredAtAccount,
    Comment,
    InstrumentID
FROM dbo.HedgeAccountTransactions WITH (NOLOCK)
WHERE LiquidityAccountID = 101
ORDER BY OccurredAtAccount DESC
```
