# Hedge.InsertManualOrderExecutionLog

> Inserts a single hedge order record into the ManualOrderExecutionLog audit table, recording both dealer-initiated manual hedge interventions and ExposureBalancer automated correction orders.

## 1. Business Meaning

`Hedge.InsertManualOrderExecutionLog` is the official write path for `Hedge.ManualOrderExecutionLog`. Every hedge order submitted outside the normal automated flow is recorded here: dealing desk operators using HedgeClient to manually intervene in hedge positions, and the ExposureBalancer service submitting automated correction orders when residual exposure imbalances are detected.

The procedure is a thin persistence layer - it maps its 14 parameters directly to the corresponding table columns and performs a single INSERT with no validation or conditional logic. It was created to separate the write concern from the caller, and @UpdateNetting was added in January 2022 (by Ran Ovadia) to capture whether the order should trigger netting recalculation.

Note: The procedure does NOT insert the newer FIX-protocol columns (OrderType, TimeInForce, ExpirationInSeconds, SlippagePercentage, ReasonID) - those are populated by a separate code path (HedgeClient direct insert). The ExecutionLogger database role holds EXECUTE permission on this procedure, indicating it is called from the hedge server's logging layer.

## 2. Business Logic

### 2.1 Manual vs Automated Order Recording

**What**: Two fundamentally different order sources write through this procedure.

**Columns/Parameters Involved**: `@Sender`, `@RequestTypeID`, `@UpdateNetting`, `@Rate`, `@TradeDescription`

**Rules**:
- **Dealing desk (manual)**: @RequestTypeID in {0-7} (Dictionary.HedgeManualRequestType values), @Sender = "HedgeClientN (by \<username\>)", @IP = internal network IP, @Rate = 0 (market order), @UpdateNetting = NULL.
- **ExposureBalancer (automated)**: @RequestTypeID = -1 (sentinel - not in dictionary), @Sender = "ExposureBalancer", @IP = NULL, @Rate = actual execution price, @UpdateNetting = 1, @TradeDescription = "Exposure Balancer Saga".
- The FK on ManualOrderExecutionLog has NOCHECK, permitting ExposureBalancer's @RequestTypeID = -1 value to bypass dictionary validation.

**Diagram**:
```
Dealing Desk Operator               ExposureBalancer Service
(via HedgeClient)                   (automated correction)
        |                                     |
@RequestTypeID = 0..7               @RequestTypeID = -1
@Sender = "HedgeClientN (by X)"     @Sender = "ExposureBalancer"
@IP = "10.x.x.x"                    @IP = NULL
@Rate = 0 (market)                  @Rate = actual market price
@UpdateNetting = NULL               @UpdateNetting = 1
@TradeDescription = NULL            @TradeDescription = "Exposure Balancer Saga"
        |                                     |
        +-----------+-----------+
                    |
     EXEC Hedge.InsertManualOrderExecutionLog(...)
                    |
     INSERT INTO Hedge.ManualOrderExecutionLog
     (OrderID, Sender, IP, ClientSendTime, HedgeStartTime,
      RequestTypeID, InstrumentID, IsBuy, AmountInUnits,
      RequestedIsBuy, RequestedAmountInUnits, TradeDescription,
      Rate, UpdateNetting)
```

## 8. Sample Queries

### 8.1 Record a manual dealing desk hedge order
```sql
EXEC [Hedge].[InsertManualOrderExecutionLog]
    @OrderID                  = NEWID(),
    @Sender                   = 'HedgeClient1 (by dealerXYZ)',
    @IP                       = '10.160.0.100',
    @ClientSendTime           = GETDATE(),
    @HedgeStartTime           = GETDATE(),
    @RequestTypeID            = 0,      -- Custom Request
    @InstrumentID             = 1,
    @IsBuy                    = 1,
    @AmountInUnits            = 100.0,
    @RequestedIsBuy           = NULL,
    @RequestedAmountInUnits   = NULL,
    @TradeDescription         = NULL,
    @Rate                     = 0,      -- market order
    @UpdateNetting            = NULL
```

### 8.2 Recent manual hedge orders from dealing desk
```sql
SELECT TOP 20
    OrderID, Sender, IP, ClientSendTime,
    RequestTypeID, InstrumentID, IsBuy, AmountInUnits, Rate
FROM [Hedge].[ManualOrderExecutionLog] WITH (NOLOCK)
WHERE RequestTypeID >= 0
ORDER BY ClientSendTime DESC
```

### 8.3 ExposureBalancer corrections in the last 7 days
```sql
SELECT InstrumentID, IsBuy, AmountInUnits, Rate, ClientSendTime, UpdateNetting
FROM [Hedge].[ManualOrderExecutionLog] WITH (NOLOCK)
WHERE RequestTypeID = -1
  AND ClientSendTime > DATEADD(day, -7, GETUTCDATE())
ORDER BY ClientSendTime DESC
```
