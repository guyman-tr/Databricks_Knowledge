## 1. Business Meaning

`Hedge.AddExpsosureAlert_Child` is the cross-database variant of `Hedge.AddExpsosureAlert`. It has an identical parameter signature and identical logic, but targets `etoro.Hedge.ExposureAlerts` using a 3-part name (database.schema.table). This means it writes to the `etoro` database's Hedge.ExposureAlerts table rather than the current database's local copy.

This variant was used when the architecture involved multiple database instances on the same SQL Server, where a child/secondary database needed to write alerts to the primary `etoro` database. The naming convention "_Child" indicates this was the "child server" version in a primary/child replication or multi-instance setup.

Per the `Hedge.ExposureAlerts` table documentation, this table is a legacy archive from 2014 and no longer actively written to. This procedure, along with its siblings (`AddExpsosureAlert` and `AddExpsosureAlert_old`), represents the evolution of the alert distribution architecture.

See `Hedge.AddExpsosureAlert` for full business context on the ExposureAlerts table and the three-variant architecture.
