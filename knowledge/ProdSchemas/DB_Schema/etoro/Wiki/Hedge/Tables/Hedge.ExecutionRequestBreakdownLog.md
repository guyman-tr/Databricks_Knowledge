## 1. Business Meaning

Hedge.ExecutionRequestBreakdownLog is the "request side" of the eToro hedge TCA (Transaction Cost Analysis) framework. Every time the hedge system sends an order to a liquidity provider, a row is inserted here capturing: the instrument, the direction and size of the request, the eToro mid-market price at that moment (from Trade.CurrencyPrice), and the provider's quoted bid/ask price at that moment.

When paired with `Hedge.ExecutionResponseBreakdownLog` (joined on HedgeID), this table enables post-trade analysis of:
- **Slippage**: How much did the execution price deviate from the price at request time?
- **Spread cost**: What was the provider's spread, and how does it compare to eToro's internal spread?
- **Price improvement/degradation**: Was execution at a better or worse price than quoted?
- **eToro margin**: The difference between eToro's price (what customers see) and the provider's mid-market rate.

This pattern powers the `Hedge.Report_TCA` SSRS report, which is the hedge desk's primary Transaction Cost Analysis tool.

The table is **empty in this environment (0 rows)**. In production it would hold execution request records. The companion `Hedge.ExecutionResponseBreakdownLog` (object #11) holds the fill-side records.

## 2. Business Logic

### 2.1 TCA Request Capture

**What**: Every hedge order request is logged with both the eToro internal price and the provider's external price, capturing the full pricing spread at request time.

**Columns/Parameters Involved**: `HedgeID`, `eToroPriceBid`, `eToroPriceAsk`, `ProviderPriceBid`, `ProviderPriceAsk`, `OccurredAtServer`

**Rules**:
- Called by `Hedge.LogHedgeExecutionRequest` on every hedge order send.
- `eToroPriceBid` and `eToroPriceAsk` are NOT passed as parameters - they are fetched at insert time via `SELECT Bid, Ask FROM Trade.CurrencyPrice WHERE InstrumentID = @InstrumentID`. This ensures the snapshot is from the live price feed at the exact moment of the request.
- `ProviderPriceBid` / `ProviderPriceAsk`: the provider's quoted price transmitted with the order request (type `dbo.dtPrice`).
- The gap between eToro prices and provider prices represents eToro's markup:
  - `eToroPriceBid - ProviderPriceBid` = eToro BID markup over provider (eToro BID slightly below provider BID)
  - `ProviderPriceAsk - eToroPriceAsk` = eToro ASK markup over provider (eToro ASK slightly above provider ASK)
- `Report_TCA` computes: `(eToroPriceBid - (ProviderPriceBid + ProviderPriceAsk) / 2)` = eToro mid vs provider mid, measuring the systematic markup captured by eToro.

### 2.2 Pairing with ExecutionResponseBreakdownLog

**What**: Each request row pairs with one or more response rows on the same HedgeID, enabling before/after price comparison.

**Columns/Parameters Involved**: `HedgeID`

**Rules**:
- HedgeID is the join key with `Hedge.ExecutionResponseBreakdownLog` in Report_TCA and InsertKPIData.
- Report_TCA uses FULL JOIN (outer join) on HedgeID to catch requests without responses (timed out) and responses without requests (edge cases).
- The time range filtering uses both `Occurred` (DB time) and `OccurredAtServer` (server time) to handle requests that straddle time window boundaries.
- `LA.LiquidityAccountID NOT IN (22,23)` in Report_TCA - excludes specific test/internal accounts from TCA analysis.
- `IsManual <> 1` in Report_TCA - excludes manually-triggered hedges from automated TCA metrics.

### 2.3 Manual vs Automated Requests

**What**: The IsManualRequest flag distinguishes system-generated hedge requests from operator-initiated ones.

**Columns/Parameters Involved**: `IsManualRequest`

**Rules**:
- IsManualRequest = 0: Automated hedge order generated by the hedge system's exposure management logic.
- IsManualRequest = 1: Manual hedge request, typically entered by operations staff for corrections or direct market intervention.
- Report_TCA filters `IsManual <> 1` to exclude manual requests from automated cost analysis, as manual orders are not subject to the same routing and pricing constraints.

### 2.4 Dual Timestamp Design

**What**: Two timestamps capture the request at two different time points.

**Columns/Parameters Involved**: `Occurred`, `OccurredAtServer`

**Rules**:
- `Occurred`: DB server UTC time at row insert. DEFAULT = GETUTCDATE(). Clustered index key for range queries.
- `OccurredAtServer`: The hedge server's own reported timestamp for when the request was generated. The authoritative request time. Range queries in Report_TCA and InsertKPIData use BOTH timestamps with different window boundaries:
  - Req.Occurred >= @StartOccurred (broader DB time range)
  - Req.OccurredAtServer >= @Start (tighter server time range)
- This dual-filter pattern prevents edge cases where a row was inserted (Occurred) slightly before/after the window but the actual request (OccurredAtServer) falls within the intended analysis window.

## 8. Sample Queries

### 8.1 eToro vs provider spread analysis (last 24h)
```sql
SELECT InstrumentID, LiquidityAccountID,
       AVG((eToroPriceBid - ProviderPriceBid) * POWER(10, 4)) AS AvgBidMarkupPips,
       AVG((ProviderPriceAsk - eToroPriceAsk) * POWER(10, 4)) AS AvgAskMarkupPips,
       COUNT(1) AS RequestCount
FROM Hedge.ExecutionRequestBreakdownLog WITH (NOLOCK)
WHERE OccurredAtServer > DATEADD(day, -1, GETUTCDATE())
  AND IsManualRequest = 0
GROUP BY InstrumentID, LiquidityAccountID
ORDER BY RequestCount DESC;
```

### 8.2 Join request and response for slippage analysis (mirrors Report_TCA)
```sql
SELECT Req.HedgeID,
       Req.InstrumentID,
       Req.OccurredAtServer AS RequestTime,
       Res.OccurredAtServer AS ExecutionTime,
       Req.IsBuy,
       DATEDIFF(ms, Req.OccurredAtServer, Res.OccurredAtServer) AS LatencyMs,
       CASE WHEN Req.IsBuy = 1
            THEN (Req.ProviderPriceAsk - Res.ExecutionPriceAsk)
            ELSE (Res.ExecutionPriceBid - Req.ProviderPriceBid) END AS SlippagePriceDiff
FROM Hedge.ExecutionRequestBreakdownLog Req WITH (NOLOCK)
JOIN Hedge.ExecutionResponseBreakdownLog Res WITH (NOLOCK)
  ON Req.HedgeID = Res.HedgeID
WHERE Req.OccurredAtServer > DATEADD(day, -1, GETUTCDATE())
  AND Req.IsManualRequest = 0
ORDER BY Req.OccurredAtServer DESC;
```
