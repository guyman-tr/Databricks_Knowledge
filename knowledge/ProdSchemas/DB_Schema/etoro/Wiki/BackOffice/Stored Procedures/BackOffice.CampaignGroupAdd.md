## 1. Business Meaning

This procedure is the creation entry point for campaign groups - the organizational containers that cluster BackOffice marketing campaigns by staff member, product segment, or marketing channel. When a BackOffice operator needs a new grouping category (e.g., a new account manager taking ownership of campaigns, or a new promotion tier being introduced), they call this procedure to register it in the system.

The procedure performs a single INSERT into `BackOffice.CampaignGroup` and returns the auto-generated `CampaignGroupID` via the OUTPUT parameter. The caller uses this ID to immediately assign campaigns to the new group via `BackOffice.Campaign.CampaignGroupID`. The Name must be unique (enforced by the `BCMG_NAME` unique index) - attempting to add a duplicate name will raise a SQL error.

## 2. Business Logic

### 2.1 Group Creation and ID Return

**What**: Inserts one row and captures the identity value for immediate use by the caller.

**Columns/Parameters Involved**: `@Name`, `@CampaignGroupID`, `BackOffice.CampaignGroup.CampaignGroupID`, `BackOffice.CampaignGroup.Name`

**Rules**:
- INSERT INTO BackOffice.CampaignGroup (Name) VALUES (@Name)
- `SCOPE_IDENTITY()` captures the new auto-incremented CampaignGroupID
- Name uniqueness enforced by BCMG_NAME unique index - duplicate name causes a SQL error (no custom error handling)
- RETURN 0 always on success (no error trapping in this procedure)
