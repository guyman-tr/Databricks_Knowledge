## 1. Business Meaning

This is an **SSDT (SQL Server Data Tools) system table**, not a business-domain table. It tracks refactoring operations - such as schema renames, table moves, and column renames - that are performed within the SSDT project and applied during database deployments. Each row represents a single refactoring operation that has been successfully applied to this database.

The table exists to ensure **idempotent deployments**. Without it, SSDT would have no way to know which refactoring operations have already been applied, potentially leading to duplicate renames or data loss during incremental deployments. The DACPAC deployment process checks this table before applying each refactoring step.

Data is written exclusively by the SSDT deployment engine (DacFx/SqlPackage) during `Publish` operations. No application code, stored procedures, or user processes read from or write to this table. It is invisible to the MoneyBusDB business domain.
