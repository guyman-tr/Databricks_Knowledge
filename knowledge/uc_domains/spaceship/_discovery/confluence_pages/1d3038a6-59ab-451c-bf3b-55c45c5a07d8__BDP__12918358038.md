# Spaceship Data transfer to eToro Datalake - Security requirements

## **Scope**

eToro has acquired Spaceship, and there is a business requirement to gather data from the Spaceship Data Lake, which is hosted in GCP, and transfer it to the eToro Azure Data Lake. Spaceship data analyst employees would need to access the data stored in eToro data lake, as well as Bing able to analyzed the combined data related to eToro/Spaceship users.

**Led by:** Data and Finance

**Goal:** Harness Spaceship data within eToro systems to unlock actionable insights and support operational excellence.
**Key Objectives:**

* Transfer critical data from Spaceship to eToro systems for:
* Enhanced analytics and reporting by BI and Finance teams.
* Daily updates for balance synchronization if included in one balance or Club features.
* Define clear data requirements and reporting structures in collaboration with relevant stakeholders.

**Expected Impact:**

* Data-driven decision-making powered by integrated insights.
* Streamlined reporting processes for operational and financial transparency.

## **Disclaimers**

1. There is no security approval for the Spaceship application to have write permissions to the eToro Data Lake. This will be planned separately in case it will be needed.
2. The scope involves the eToro data team initiating read requests to collect data and transfer it securely to the eToro Data Lake.
3. No data will be collected directly from eToro data platform to Spaceship Data Lake. The plan is to use API calls from Spaceship platform that will be served by eToro APIs that will be exposed via eToro API MNG. This is not covered under this documents.
4. **PCI data cant be transferred** directly from Spaceship to eToro and cant be stored in eToro Data lake. need to involve Security Architect (Yafit Avichzer, deactivated) and Security GRC (Israela Yukelis) to plan this activity any activity involved with PCI data in advance.

## **Security guidelines**

### **PII Separation**

The Spaceship team needs to identify all columns containing PII (Personally Identifiable Information) and provide the column names to Pini's team in advance. The eToro team will deliver the data in two separate schemas: one containing PII fields and the other without PII, **avoid any PCI data in any case.**

The data coming from Spaceship will be integrated into the eToro schema (not in a separate container) and will go through the same data platform process of Bronze → Silver → Gold.

### **Spaceship access to eToro Data platform**

Will be using eToro users based on eToro AAD. access permissions will be granted according to business role like we have for eToro Data employs. PII access permissions via Fresh service ticket.

### **Collecting data from Spaceship GCP lake**

To transfer data from a Google Cloud Platform (GCP) data lake to an Azure Storage account data lake, you should use Azure Data Factory. Azure Data Factory provides connectors that allow you to copy data from Google Cloud Storage to Azure Data Lake Storage.

1. **Setup Azure Data Factory**: Create an instance of Azure Data Factory in your Azure account.
2. **Create Linked Services**: Set up linked services in Azure Data Factory for both Google Cloud Storage and Azure Data Lake Storage.
3. **Copy Data Activity**: Use the Copy Data tool in Azure Data Factory to define a copy activity that specifies the source (Google Cloud Storage) and the destination (Azure Data Lake Storage).
4. **Mapping Data Flows**: If you need to transform the data during the transfer, you can use mapping data flows to read and transform the data as needed.
5. **Run the Pipeline**: Once everything is set up, you can run the pipeline to initiate the data transfer.
6. Ensure that you have the necessary permissions set up in both Google Cloud and Azure environments to facilitate the data transfer.
7. **Transport Layer Security (TLS)**: Azure uses TLS to protect data while it is in transit between cloud services and customers. Ensure that your applications are configured to use TLS for secure communication.
