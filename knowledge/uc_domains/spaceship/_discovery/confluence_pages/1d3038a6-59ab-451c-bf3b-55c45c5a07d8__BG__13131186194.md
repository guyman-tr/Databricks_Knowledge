# SpaceShip Integration - LLD

## 1. General — Background

eToro and Spaceship will exist as **independent but connected platforms**. Spaceship will remain independent to eToro because Spaceship will serve a different target market to eToro. They will be connected because:

* there will be cross-sell opportunities between the two platforms and we want the customer to be able to explore, view and transact (to an extent) on each other's products;
* there will be a common cash account to be available to use on both platforms; and
* we want Spaceship assets to contribute towards eToro Club benefits.

This independent but connected strategy will maximize our market share of investors in Australia so that we become the largest self-directed investment house in Australia.

### 5.1 Dependencies

* Eligibility API (Get SpaceshipEligibilityOption)

### 5.2 Exposed API's

#### WalletTab BE

`POST /api/etwallet/v1/balances` — extended with a SpaceShip section.

#### BalanceAdapter.SpaceShip

`GET /api/v1/balances/{gcid}` — new endpoint for communication and rearranging balances data from SpaceShipAPI.

#### Example — BalanceAdapter.Spaceship response

```json
{
  "LearnMoreLink": "https://spacehip.com???",
  "AccountLink": "https://spaceship.com????",
  "balances": [
    { "AssetId": 2, "DisplayName": "US Investing",   "CurrencyISO": "AUD", "Amount": 12345.6478, "PendingAmount": 100.00 },
    { "AssetId": 1, "DisplayName": "Available money","CurrencyISO": "AUD", "Amount": 500000.00, "PendingAmount": 200.00 }
  ]
}
```

#### Example — SpaceshipAPI response

```json
{
  "userId": "abc123",
  "balances": [
    { "ProductId": 1, "BalanceType": "CASH",        "DisplayingName": "Available money", "MarketValue": 500000.00,  "PendingInvestmentValue": 200, "CurrencyISO": "AUD", "LastUpdated": "2025-04-09T03:15:00Z" },
    { "ProductId": 2, "BalanceType": "US_INVESTING","DisplayingName": "US Investing",    "MarketValue": 1200000.50, "PendingInvestmentValue": 100, "CurrencyISO": "AUD", "LastUpdated": "2025-04-09T03:15:00Z" },
    { "ProductId": 3, "BalanceType": "SUPER",       "DisplayingName": "Super",           "MarketValue": 875000.25,  "PendingInvestmentValue": 100, "CurrencyISO": "AUD", "LastUpdated": "2025-04-09T03:15:00Z" },
    { "ProductId": 4, "BalanceType": "VOYAGER",     "DisplayingName": "Voyager",         "MarketValue": 875000.25,  "PendingInvestmentValue": 0,   "CurrencyISO": "AUD", "LastUpdated": "2025-04-09T03:15:00Z" }
  ]
}
```

The four `BalanceType` values are the canonical Spaceship product enum: **CASH** (Available money), **US_INVESTING**, **SUPER** (superannuation/retirement), **VOYAGER** (managed funds). All amounts are in AUD by source.

## 7.1 [BE] [BalanceAdapter.SpaceShip] — Get Balances infrastructure

In order to start implementing SpaceShip integration we need to create a fully working new service, with all necessary functionality for receiving balances:

* Add functionality for acquiring authorization token
* Add REST API call `GET /api/v1/balances/{gcid}`
* Add functionality for receiving balance information from third party
* Add Mocks for temporary accessibility while other teams preparing APIs
