# MoneyFarm - global payments configurations

## MoneyBus DB

```sql
USE [MoneyBusDB]
GO

-- Assume the Identity Column is named [AccountTypeID] based on standard naming conventions.
-- We check for the existence of the record using the hardcoded ID (4).
IF NOT EXISTS (
    SELECT 1
    FROM [Dictionary].[AccountTypes]
    WHERE [ID] = 4
)
BEGIN
    SET IDENTITY_INSERT [Dictionary].[AccountTypes] ON;

    INSERT INTO [Dictionary].[AccountTypes]
        ([ID], [Name])
    VALUES
        (4, 'MoneyFarm');

    SET IDENTITY_INSERT [Dictionary].[AccountTypes] OFF;

    PRINT 'Successfully inserted AccountTypeID 4 (MoneyFarm).'
END
ELSE
BEGIN
    PRINT 'AccountTypeID 4 (MoneyFarm) already exists. No action taken (Idempotent).'
END
GO
```

## CCM

| PaymentsAggregatedEvents | AccountTypeNames | add "4":"MoneyFarm" |
| CustomerFinanceService | ValidUnifiedFtdAccountTypes | add 4 |

## Key takeaways for data domain

* **AccountTypeID = 4 = MoneyFarm** in all eToro payment ledgers (etoro DB, MoneyBusDB, CustomerFinanceDB). Combined with `bi_db.bronze_sub_accounts_accounts`, this is the join key from the eToro side to MoneyFarm transactions.
* **FundingTypeID = 44 = MoneyFarm** in `Dictionary.FundingType` (DefaultCurrency=5).
* `PaymentsAggregatedEvents.AccountTypeNames` config maps `4 -> MoneyFarm`.
* `CustomerFinanceService.ValidUnifiedFtdAccountTypes` includes `4`, which is what powers MIMO FTD lookups for the v_moneyfarm_mimo view.
