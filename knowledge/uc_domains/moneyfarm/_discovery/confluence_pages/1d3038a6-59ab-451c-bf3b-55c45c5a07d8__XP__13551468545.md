# MF additions - Support for Money Farm Deposit Event

_pageId=13551468545 url=https://etoro-jira.atlassian.net/wiki/spaces/XP/pages/13551468545/MF+additions+-+Support+for+Money+Farm+Deposit+Event_

| **Term/Acronym** | **Definition** |
|---|---|
| MF | Money Farm |
| FTD | First Time Deposit |
| EH | Event Hub |
| SA | Sub Account |
| SB | Service Bus |
| MOP | Mean Of Payment |

## General

### Background

The goal of this document is to describe the changes to the Sub Account System in order to support new Status that indicate that user is also onboarded and deposited.
[sub-accounts-experience-worker](https://github.com/eToro/sub-accounts-experience-worker) service will perform the following operations:

1. Listen to MF deposit events from event-streaming EH (this EH stores all sub account incoming messages, owned by Infra team).
2. Call Payments CustomerFinance to get deposits (CustomerFinance WebApi).
3. Create transaction ID by Hash using: `GCID`, `valueDate`, `Amount`.
4. Publish new Deposit Event to Mimo Service Bus.

## General Flow

1. Listing to Infra EH `event-streaming` under `sub-accounts` Entity. See Event Details (sample):

```json

    "eventType": "PORTFOLIO_DEPOSIT",
    "createdAt": "2025-09-18T10:30:45"
  },
  "eventData": {

```


2. Call CustomerFinance WebApi to get FTD for accounts: `/api/v{version}/Customers/{gcid}/first-time-deposit/accounts`. If no Deposit, set `IsFTD = true`.
3. Generate `TransactionID`: hash of (`GCID`, `valueDate`, `Amount`).
4. Publish new Event to `{env}-payments-metrics` SB with structure:

```json
{
  "GCID": 11157065,
  "AccountTypeId": 1,
  "AccountId": "14540678",
  "CreatedDate": "2025-09-14T10:05:08Z",
  "ModifiedDate": "2025-09-14T10:05:07Z",
  "TransactionId": "4783212",
  "ExTransactionId": "4e7dc7ff246b44af9907",
  "InputAmount": { "Amount": 1000, "Currency": "AUD" },
  "FeeAmount": { "Amount": 15, "Currency": "USD" },
  "PaymentMethodTypeId": 37,
  "PaymentMethodId": "3584856",
  "PaymentStatusId": 1,
  "DepositTypeId": 0,
  "ReferenceId": "",
  "IsFtd": false,
  "AmountInUsd": 663.8,
  "ExtraData": {}
}
```

`AccountTypeId` legend: 1=Trading, 2=Options, 3=IBAN, **4=MoneyFarm** (this page).
`PaymentMethodTypeId` legend: 2=Wire, 29=ACH, 38=OpenBanking, 42=eToroOptions, **44=MoneyFarm** (per global-payments-config page).

## Deposit event Mapping (MoneyFarm specifics)

| **Property** | **Value** | **Notes** |
|---|---|---|
| `GCID` | `GCID` | Source |
| `AccountTypeId` | `4` | hard coded value |
| `AccountId` | `GCID` | Infra Event |
| `CreatedDate` | `ValueDate` | Infra Event |
| `ModifiedDate` | `ValueDate` | Infra Event |
| `TransactionId` | new created hash(GCID, valueDate, Amount) |  |
| `ExTransactionId` | same value as `TransactionId` or empty string | Infra Event |
| `Input.Amount` | `amount` | Infra Event |
| `Input.Currency` | `currency` | Infra Event |
| `FeeAmount` | null |  |
| `PaymentMethodTypeId` | `44` | Hard-coded, stored in CCM |
| `PaymentStatusId` | `2` | Success |
| `DepositTypeId` | `2` |  |
| `ReferenceId` | empty string |  |
| `IsFtd` | true/false from CustomerFinance WebApi |  |
| `AmountInUsd` | null |  |
| `ExtraData` | empty or null |  |
