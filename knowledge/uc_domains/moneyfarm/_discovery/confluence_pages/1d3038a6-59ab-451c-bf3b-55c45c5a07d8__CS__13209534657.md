# Individual Savings Account (ISA) - MoneyFarm (For CS TLs)

_pageId=13209534657 url=https://etoro-jira.atlassian.net/wiki/spaces/CS/pages/13209534657/Individual+Savings+Account+ISA+-+MoneyFarm+For+CS+TLs_

## Internal guidelines

As per the CS procedure [Individual Savings Account (ISA) - MoneyFarm](https://etoro-jira.atlassian.net/wiki/spaces/CS/pages/11942330382), you may encounter an open Tmail to handle:

## CS TLs procedure

1. Review the Tmail description and case to understand what information or issue the client has.
   1. If the timeframe for a deposit or transfer has passed, send client to Moneyfarm.
2. Ask in Teams **ISA-Support** chat.
3. Get the Moneyfarm External ID from the [Tableau report](https://reports.etorocorp.com/#/views/ISACustomerLookupDashboard/ISACustomerDashboard?%3Aiid=1) ("ISACustomerLookupDashboard").
4. Client-specific queries need to be added to the [ISA Questions Monday board](https://etoro.monday.com/boards/18408760798/) using this [form](https://forms.monday.com/forms/0df1deac317a5c5579dcee5dcb92fb42?r=use1).
5. Once there is a reply/solution/information from MoneyFarm, solve the Tmail.

NOTE: Generic questions (ones where notebook returns nothing so say contact moneyfarm) can be sent directly to Stephanie, Spencer, or Sivan to be added to the Q and A.

## Why this matters for the data domain

The CS workflow leans on a Tableau dashboard called **ISACustomerLookupDashboard / ISACustomerDashboard** for MoneyFarm External ID lookups. This is direct evidence that at least one Tableau workbook exists in the eToro reports server that touches MoneyFarm customer identifiers — even if the global Tableau extract under `knowledge/tableau/_index/` shows no fact-table-level usage. The dashboard likely sources from `bi_output.bi_output_moneyfarm_customers` (which has a `Moneyfarm External ID` style column under `MF_Journey_Beginning`/`Date_Source_Type`).
